Section 1

What is a Neural Network?

1.1 Biological Inspiration

The artificial neural network is loosely inspired by how the human brain processes information. The brain contains approximately 86 billion neurons, each connected to thousands of others through synapses. Learning happens when the strength of these connections changes.

  • ▸
    Biological Neuron: Receives signals through dendrites, processes them in the cell body (soma), and fires an output through the axon if the total signal crosses a threshold.
  • ▸
    Artificial Neuron (Perceptron): Receives numerical inputs, multiplies each by a weight, sums them all, adds a bias, and passes the result through an activation function to produce an output.
  • ▸
    Synapse → Weight: The strength of a biological synaptic connection is modeled by the weight w in an artificial neuron. Learning = adjusting weights.

1.2 The Artificial Neuron (Perceptron)

An artificial neuron is the basic computational unit. It performs two operations: a linear transformation followed by a non-linear activation.

  x₁ ──(w₁)──┐
  x₂ ──(w₂)──┤
  x₃ ──(w₃)──┼──→  z = Σ(wᵢxᵢ) + b  ──→  a = f(z)  ──→  Output
   ⋮         │
  xₙ ──(wₙ)──┘         + bias (b)
Figure 1.1 — Single artificial neuron. Inputs are weighted, summed, biased, then activated.
Mathematical form of a single neuron
z = w₁x₁ + w₂x₂ + ... + wₙxₙ + b → weighted sum + bias
a = f(z) → activation output
  • ▸
    x₁, x₂, ..., xₙ — Input features (the data given to the neuron).
  • ▸
    w₁, w₂, ..., wₙ — Weights. These are learnable parameters. They determine how important each input is.
  • ▸
    b — Bias. A constant added to shift the output. It lets the neuron activate even when all inputs are zero.
  • ▸
    z — Pre-activation (or net input). The raw linear combination before applying the activation function.
  • ▸
    f(z) — Activation function. Introduces non-linearity. Without this, stacking multiple layers gives the same result as one layer.
  • ▸
    a — Post-activation (the output of the neuron, passed to next layer).

1.3 Activation Functions

Activation functions introduce non-linearity into the network, allowing it to learn complex patterns. Without non-linearity, any deep network collapses into a single linear transformation.

FunctionFormulaOutput RangeUse Case
Sigmoid (σ) 1 / (1 + e⁻ᶻ) (0, 1) Binary classification output layer
Tanh (eᶻ − e⁻ᶻ) / (eᶻ + e⁻ᶻ) (−1, 1) Hidden layers (zero-centered, better than sigmoid)
ReLU max(0, z) [0, ∞) Most hidden layers (fast, avoids vanishing gradient)
Leaky ReLU max(0.01z, z) (−∞, ∞) Avoids "dying ReLU" problem
Softmax eᶻᵢ / Σeᶻⱼ (0, 1), sums to 1 Multi-class classification output layer
Key Insight — Why Non-Linearity?

A composition of linear functions is still linear: f(g(x)) = f(Ax) = A'x. So without activation functions, a 100-layer network can only learn what a 1-layer network can. Non-linearity is what gives deep networks their power.

Vanishing Gradient Problem (Preview)

Sigmoid and Tanh squash their inputs into small ranges. Their derivatives are very small (Sigmoid max = 0.25). When chained across many layers during backpropagation, gradients shrink to near zero — the network stops learning in early layers. ReLU largely solves this.

1.4 Network Architecture — Layers

  INPUT LAYER    HIDDEN LAYER 1   HIDDEN LAYER 2   OUTPUT LAYER
  ┌───────┐       ┌───────┐        ┌───────┐        ┌───────┐
  │  x₁   │──────▶│  h₁₁  │───────▶│  h₂₁  │───────▶│  ŷ₁  │
  │  x₂   │──────▶│  h₁₂  │───────▶│  h₂₂  │───────▶│  ŷ₂  │
  │  x₃   │──────▶│  h₁₃  │───────▶│  h₂₃  │        └───────┘
  └───────┘       │  h₁₄  │        └───────┘
                  └───────┘
  (no learning)   (learnable)       (learnable)      (learnable)
Figure 1.2 — Fully connected (dense) neural network with 2 hidden layers.
  • ▸
    Input Layer: Receives the raw features. No computation is done here. The number of neurons = number of features in your data.
  • ▸
    Hidden Layers: Intermediate layers where all computation happens. Each neuron in a hidden layer applies the weighted sum + activation. These layers learn abstract representations of data.
  • ▸
    Output Layer: Produces the final prediction. Number of neurons depends on the task (1 for regression/binary, K for K-class classification).
  • ▸
    Fully Connected (Dense): Every neuron in one layer connects to every neuron in the next layer. This is the standard architecture.
Section 2

How a Neural Network Works — Forward Propagation

Before a network can learn, it must first make a prediction. This prediction process — passing input data through the network layer by layer — is called Forward Propagation.

2.1 Forward Propagation — Step by Step

  1. Initialize weights and biases — Each connection has a weight w and each neuron has a bias b. Initially, these are set to small random values (not zero, to break symmetry).
  2. Feed input data — Provide input vector x = [x₁, x₂, ..., xₙ] to the input layer. These propagate to the first hidden layer.
  3. Compute pre-activation z for each layer — For layer l: z[l] = W[l] · a[l-1] + b[l], where W[l] is the weight matrix and a[l-1] is the output of the previous layer.
  4. Apply activation function — a[l] = f(z[l]). This becomes the input to the next layer.
  5. Repeat for each layer — Carry this forward through every hidden layer until the output layer.
  6. Compute final output ŷ — The output layer produces the prediction ŷ. For classification, typically a Sigmoid (binary) or Softmax (multi-class) is applied.
  7. Compute the Loss (Cost) — Compare ŷ to the true label y using a loss function to measure the error.

2.2 Loss Functions

The loss function measures how wrong the prediction is. The goal of training is to minimize this loss.

Binary Cross-Entropy Loss (Binary Classification)
L(ŷ, y) = −[ y · log(ŷ) + (1−y) · log(1−ŷ) ]
Mean Squared Error Loss (Regression)
L(ŷ, y) = (1/2) · (ŷ − y)²
Cost Function (average loss over all m training examples)
J(W, b) = (1/m) · Σᵢ L(ŷ⁽ⁱ⁾, y⁽ⁱ⁾)
  • ▸
    Loss L: Error for a single training example.
  • ▸
    Cost J: Average loss over the entire training dataset. The network tries to minimize J.
  • ▸
    Why Cross-Entropy for classification? It heavily penalizes confident wrong predictions. If the model says "90% chance of class 1" but the answer is 0, the penalty is −log(0.1) ≈ 2.3 — very large.
Intuition — What is the network doing?

Think of a network as a function: f(x; W, b) → ŷ. The weights and biases are the parameters. Forward propagation is just evaluating this function. Training is finding the values of W and b that make this function approximate the true mapping from x to y as closely as possible.

Section 3

Gradient Descent

After computing the loss, the network needs to update its weights to reduce the error. Gradient Descent is the optimization algorithm used to do this. It uses the gradient (slope) of the cost function to decide in which direction to move the weights.

3.1 Core Idea — The Mountain Analogy

Intuition

Imagine you are standing somewhere on a hilly landscape (the cost surface) blindfolded. Your goal is to reach the lowest valley (minimum cost). At each step, you feel the slope beneath your feet (gradient) and take a step in the direction that goes downhill. That is gradient descent.

3.2 The Update Rule

Gradient Descent Parameter Update
W := W − α · (∂J/∂W) ← update weights
b := b − α · (∂J/∂b) ← update biases
  • ▸
    α (learning rate): A small positive number (e.g. 0.01). Controls how large each step is. Too large → overshoot the minimum. Too small → training is very slow.
  • ▸
    ∂J/∂W (gradient): The partial derivative of the cost function with respect to the weights. It tells us: "if I increase W slightly, how much does J increase?" If this is positive, J increases with W, so we decrease W to minimize J (hence the minus sign).
  • ▸
    Why partial derivative? J depends on all weights simultaneously. Partial derivative isolates the effect of changing one weight while holding others fixed.
  • ▸
    := notation: This means "assign", i.e., W is being updated in place — old W replaced by new value.

3.3 Variants of Gradient Descent

VariantWhat it uses per updateSpeedStability
Batch GD All m training examples Slow per step (but accurate direction) Very stable, smooth convergence
Stochastic GD (SGD) 1 training example at a time Very fast per step Noisy, oscillates; can escape local minima
Mini-Batch GD Small batch (e.g. 32, 64, 128 examples) Fast, vectorized on GPU Good balance — used in practice
Which variant is used in practice?

Mini-batch gradient descent (typically batch size 32–256) is the standard in deep learning. It exploits vectorized operations on GPUs/TPUs while still updating more frequently than full batch GD. When people say "SGD" in deep learning, they usually mean mini-batch SGD.

3.4 The Role of the Learning Rate

  • ▸
    Too high α: Overshoots the minimum. Cost may oscillate or diverge (get worse over time).
  • ▸
    Too low α: Learning is painfully slow. May get stuck in shallow local minima.
  • ▸
    Just right α: Cost decreases smoothly and reaches a minimum in a reasonable number of iterations.
  • ▸
    Learning rate decay: Start with a larger α and reduce it over time. Common approach: α decreases as training progresses, allowing fine-tuning near the minimum.

3.5 Local vs Global Minima

  • ▸
    Global minimum: The absolute lowest point of the cost function. Theoretically ideal but hard to guarantee finding.
  • ▸
    Local minimum: A low point that is not the lowest overall. GD can get trapped here.
  • ▸
    Saddle points: Points where gradient is zero but it is neither a max nor a min. More common than local minima in high-dimensional spaces. GD can slow down here.
  • ▸
    In practice: For deep networks with millions of parameters, most local minima are actually pretty good — almost as good as the global minimum (Goodfellow et al.). So GD usually finds a good solution.
Exam Point — Convergence Condition

Gradient descent is said to have converged when the change in cost J between successive iterations becomes negligibly small (below a threshold ε). Alternatively, when ‖∇J‖ ≈ 0, meaning we are at a flat region of the cost surface.

Section 4

Backpropagation Algorithm

Gradient descent requires us to know the gradient ∂J/∂W for every weight in the network. For a deep network with millions of parameters, computing this directly is infeasible. Backpropagation is the efficient algorithm that computes all these gradients in one backward pass using the chain rule of calculus.

4.1 The Chain Rule — The Heart of Backpropagation

Chain Rule (Calculus Recap)

If z depends on y, and y depends on x, then the derivative of z with respect to x is: dz/dx = (dz/dy) · (dy/dx). In a neural network, the loss J depends on the output, which depends on the last layer, which depends on the layer before it — all the way back to the input. The chain rule lets us unravel this dependency.

4.2 Notation for Backpropagation

  • ▸
    L — Total number of layers (not counting input).
  • ▸
    a[l] — Activations (outputs) of layer l.
  • ▸
    z[l] — Pre-activation values of layer l: z[l] = W[l]·a[l-1] + b[l].
  • ▸
    δ[l] (delta) — The error signal at layer l. Formally: δ[l] = ∂J/∂z[l]. This is the sensitivity of the cost to the pre-activation at layer l.
  • ▸
    f'(z) — Derivative of the activation function (needed for gradient computation).

4.3 Step-by-Step Backpropagation

  1. Forward pass first — Run forward propagation to compute all z[l] and a[l] for all layers. Also compute the loss J. Store all intermediate values — they are needed during backprop.
  2. Compute output layer error δ[L]
    δ[L] = ∂J/∂z[L] = (∂J/∂a[L]) · f'(z[L])
    For Binary Cross-Entropy with Sigmoid output: δ[L] = (a[L] − y)
  3. Propagate error backwards — for each layer l = L-1, L-2, ..., 1
    δ[l] = (W[l+1]ᵀ · δ[l+1]) ⊙ f'(z[l])
    Where ⊙ is element-wise multiplication and W[l+1]ᵀ is the transpose of next layer's weight matrix.
  4. Compute gradients for weights and biases at each layer
    ∂J/∂W[l] = δ[l] · a[l-1]ᵀ ← gradient w.r.t. weights
    ∂J/∂b[l] = δ[l] ← gradient w.r.t. biases
  5. Update parameters using gradient descent
    W[l] := W[l] − α · ∂J/∂W[l]
    b[l] := b[l] − α · ∂J/∂b[l]
  6. Repeat for many iterations (epochs) until the cost converges.

4.4 Intuition — What is Backprop Really Doing?

  • ▸
    Blame assignment: Backpropagation answers: "Which weights were responsible for the error, and by how much?" The gradient ∂J/∂w tells us — a large gradient means that weight contributed significantly to the error.
  • ▸
    Efficiency: A naive approach would compute the gradient for each weight independently, requiring one forward pass per parameter. Backprop computes ALL gradients in just two passes (one forward, one backward) — this is its genius.
  • ▸
    Why backward? The error starts at the output (where we know the loss) and flows backwards because that is the direction of the chain rule — the loss depends on output, which depends on the second-to-last layer, etc.
  FORWARD PASS (compute predictions)
  x ──→ [Layer 1] ──→ [Layer 2] ──→ [Layer 3] ──→ ŷ ──→ LOSS J
              ↑              ↑              ↑
         store z[1],a[1]  store z[2],a[2]  store z[3],a[3]

  BACKWARD PASS (compute gradients)
  J ──→ δ[3] ──→ δ[2] ──→ δ[1] ──→ (done)
         ↓           ↓          ↓
      ∂J/∂W[3]   ∂J/∂W[2]   ∂J/∂W[1]
Figure 4.1 — Forward and backward passes through a 3-layer network.
Exam Point — Key Properties of Backpropagation

1. Backpropagation is not an optimization algorithm — it only computes gradients. Gradient Descent uses those gradients to update parameters.

2. Backprop requires storing all intermediate activations from the forward pass — this is why deep networks use a lot of memory during training.

3. Backpropagation is just the chain rule applied efficiently using dynamic programming.

4.5 Derivative of Common Activation Functions

These are needed in the backpropagation step when computing δ[l] = ... ⊙ f'(z[l]).

Activation Derivatives
Sigmoid: σ'(z) = σ(z) · (1 − σ(z)) ← max value is 0.25 at z=0
Tanh: tanh'(z) = 1 − tanh²(z) ← max value is 1 at z=0
ReLU: ReLU'(z) = 1 if z > 0, else 0 ← constant gradient, no squashing
Section 5

Introduction to Deep Neural Networks

5.1 What Makes a Network "Deep"?

  • ▸
    Shallow Network: 1 hidden layer. Can theoretically approximate any function (Universal Approximation Theorem), but requires exponentially many neurons.
  • ▸
    Deep Network: 2 or more hidden layers. In practice, "deep" usually means many hidden layers (5, 10, 100+). Deep Learning = learning with deep neural networks.
  • ▸
    Depth matters: More layers mean the network can build increasingly abstract representations of data hierarchically.

5.2 Hierarchical Feature Learning

The power of deep networks comes from their ability to learn a hierarchy of features. Each layer transforms the representation from the previous layer into something more abstract and task-relevant.

  Example: Image Recognition (Face Detection)

  Layer 1 → Detects edges, corners, gradients
  Layer 2 → Combines edges into shapes: eyes, nose, curves
  Layer 3 → Combines shapes into face parts: eye region, jaw
  Layer 4 → Detects whole face structure
  Layer 5 → Recognizes specific people, expressions

  Low-level features ──────────────────────────→ High-level abstractions
Figure 5.1 — Hierarchical feature learning in a deep network for face recognition.
  • ▸
    Early layers learn low-level, general features (edges, textures). These are similar across different tasks.
  • ▸
    Middle layers learn intermediate, task-specific features by composing low-level ones.
  • ▸
    Later layers learn high-level, abstract, task-specific representations directly useful for classification/regression.
  • ▸
    Transfer learning exploits this: early layers trained on one task (e.g. ImageNet) can be reused for another task because low-level features are universal.

5.3 Why Deep Works Better than Wide

  • ▸
    Parameter efficiency: A deep network can represent certain functions exponentially more compactly than a shallow one. This means fewer total parameters needed for the same expressiveness.
  • ▸
    Circuit complexity theory: There exist functions that require exponential neurons in a shallow network but only polynomial neurons in a deep network (Bengio & LeCun).
  • ▸
    Compositionality of real world: Real-world data (images, speech, text) is naturally compositional — objects are made of parts, parts are made of primitives. Deep networks mirror this structure.

5.4 Challenges in Deep Networks

Vanishing Gradients

Gradients shrink exponentially through many sigmoid/tanh layers. Early layers learn very slowly or not at all.

Exploding Gradients

Gradients grow exponentially through layers. Weights update wildly; training becomes unstable. Fixed with gradient clipping.

Overfitting

Deep networks have many parameters and can memorize training data. Solved with dropout, L2 regularization, batch norm.

Computational Cost

Training deep networks requires large datasets and significant compute (GPUs/TPUs). Inference can also be slow.

Universal Approximation Theorem

A single hidden layer neural network with a sufficient number of neurons can approximate any continuous function on a compact domain to arbitrary precision. However, "sufficient" can mean exponentially many neurons. Deep networks achieve the same approximation with polynomially fewer neurons.

Section 6

Role of Vectorization in Deep Learning

In deep learning, we process thousands of training examples and layers with potentially millions of neurons. Writing operations as explicit loops is extremely slow. Vectorization is the practice of replacing loops with matrix/vector operations that execute in parallel on modern hardware.

6.1 What is Vectorization?

  • ▸
    Scalar approach: Process one data point at a time using for loops. Very slow for large datasets.
  • ▸
    Vectorized approach: Express all operations as matrix multiplications and element-wise operations. Entire batches processed simultaneously.
  • ▸
    Why fast? BLAS (Basic Linear Algebra Subroutines) libraries and GPUs are optimized for matrix operations using parallelism — thousands of cores compute simultaneously.
  • ▸
    In NumPy/PyTorch/TensorFlow: A single function call like np.dot(W, X) runs optimized C/CUDA code under the hood, far faster than any Python loop.

6.2 Non-Vectorized vs Vectorized Comparison

Without Vectorization (Slow — Python loops)
# m training examples, n features # Forward pass: z = W·x + b for each example z = np.zeros(m) for i in range(m): for j in range(n): z[i] += W[j] * X[j, i] z[i] += b # Time: O(m × n) sequential operations
With Vectorization (Fast — Matrix Operation)
# X is (n × m) matrix — all m examples at once # W is (1 × n) vector, b is a scalar Z = np.dot(W, X) + b # Shape: (1 × m) A = sigmoid(Z) # Element-wise — all m outputs at once # One line replaces nested loops! # Runs in parallel on all available CPU/GPU cores

6.3 Matrix Dimensions in Neural Networks

Getting matrix dimensions right is critical. Here is the complete set of rules:

QuantityDimensionsExplanation
Input X (nₓ × m) nₓ features, m training examples
Weight matrix W[l] (n[l] × n[l-1]) n[l] neurons in layer l, n[l-1] in previous layer
Bias vector b[l] (n[l] × 1) One bias per neuron in layer l
Pre-activation Z[l] (n[l] × m) Z[l] = W[l] · A[l-1] + b[l]
Activation A[l] (n[l] × m) A[l] = f(Z[l]) — same shape as Z[l]
Gradient dW[l] (n[l] × n[l-1]) Same shape as W[l]
Gradient db[l] (n[l] × 1) Same shape as b[l]
Memory Aid for Dimensions

For weight matrix W[l]: think of it as a transformation from the previous layer's space (n[l-1] dimensions) to the current layer's space (n[l] dimensions). So rows = current layer neurons, columns = previous layer neurons: W[l] is (n[l] × n[l-1]).

6.4 Broadcasting

  • ▸
    Definition: When performing operations between arrays of different shapes, NumPy/PyTorch automatically "broadcasts" the smaller array to match the shape of the larger one — without actually copying data.
  • ▸
    Example: Adding bias b (shape n[l] × 1) to Z (shape n[l] × m): the bias is automatically added to every column (every training example) — no loop needed.
  • ▸
    Why important: Broadcasting is used constantly in forward and backward passes. Understanding it prevents shape mismatch bugs.

6.5 Vectorized Forward and Backward Propagation

Vectorized Forward Propagation (entire training set)
A[0] = X ← input (nₓ × m)
Z[l] = W[l] · A[l-1] + b[l] ← (n[l] × m)
A[l] = f(Z[l]) ← element-wise activation
J = (1/m) · Σ L(A[L], Y) ← average cost over all m examples
Vectorized Backward Propagation
dZ[L] = A[L] − Y ← output error (for cross-entropy + sigmoid)
dW[l] = (1/m) · dZ[l] · A[l-1]ᵀ ← weight gradient
db[l] = (1/m) · np.sum(dZ[l], axis=1) ← bias gradient (sum over m examples)
dZ[l-1] = W[l]ᵀ · dZ[l] ⊙ f'(Z[l-1]) ← propagate error backwards
Practical Tip — Always Verify Shapes

The most common bug in implementing neural networks from scratch is a shape mismatch in matrix multiplication. Always check: (n[l] × n[l-1]) · (n[l-1] × m) = (n[l] × m). If your shapes don't follow this rule, something is wrong.

Section 7

Shallow vs Deep Networks — Comparison

Understanding why depth matters — and when it does not — is central to both theory and practice in neural networks.

7.1 Definitions Recap

  • ▸
    Shallow Network: Typically refers to a network with just 1 hidden layer. Sometimes "shallow" means 1–2 hidden layers. A logistic regression is the extreme case — 0 hidden layers.
  • ▸
    Deep Network: Has many hidden layers (typically 3 or more). Modern deep learning architectures have dozens to hundreds of layers (e.g. ResNet-152 has 152 layers).

7.2 Head-to-Head Comparison

Aspect Shallow Network Deep Network
Number of layers 1 hidden layer Many hidden layers (2, 5, 100+)
Feature learning Learns flat features directly from input Learns hierarchical, compositional features
Parameter efficiency Requires exponentially many neurons for complex functions Can represent same functions with far fewer total parameters
Training difficulty Easy to train (no vanishing gradient issue) Harder: vanishing gradients, needs careful initialization
Generalization May not generalize well on complex tasks Better generalization on complex real-world data (with regularization)
Computation Faster training and inference Slower (more matrix ops, more memory)
Data requirements Works with smaller datasets Typically needs large datasets to shine
Use cases Simple tabular data, small problems Images, speech, NLP, complex tasks
Expressiveness Limited by width (number of neurons) Highly expressive — both depth and width contribute
Interpretability Relatively easier to interpret Very hard to interpret (black box)

7.3 The Depth-Width Trade-off

  • ▸
    Width: More neurons per layer → richer representations at each level of abstraction, but no hierarchical composition.
  • ▸
    Depth: More layers → allows composing simple functions into complex ones. Each layer reuses the representations learned by the previous layer.
  • ▸
    Theoretical result: For parity functions and other complex Boolean functions, a deep network needs O(n) neurons while a shallow network needs O(2ⁿ) neurons (exponential). Depth wins dramatically.
  • ▸
    Practical wisdom: Going deeper (adding layers) generally works better than going wider (adding neurons per layer) for most real-world tasks, up to a point — beyond which we get diminishing returns and training instability.

7.4 When Shallow Networks are Sufficient

  • ▸
    Simple, low-dimensional data: Small tabular datasets with clear features. A 1-hidden-layer network or even logistic regression may be optimal.
  • ▸
    Small training data: Deep networks need lots of data to generalize. With little data, they overfit. Shallow networks are more data-efficient.
  • ▸
    When interpretability matters: In medical or financial domains, shallow models are preferred because their decisions can be explained.
  • ▸
    Edge/embedded deployment: When compute is limited (mobile devices, IoT sensors), a small shallow network is preferred.

7.5 When Deep Networks Excel

  • ▸
    Images: Convolutional deep networks (CNNs) learn edge → shape → object hierarchies. Shallow networks cannot.
  • ▸
    Speech and audio: Raw waveforms have hierarchical structure (samples → phonemes → words → sentences).
  • ▸
    Natural language: Characters → words → phrases → sentences → meaning. Deep Transformers (BERT, GPT) exploit this hierarchy.
  • ▸
    Any structured compositional data: Where the data itself is naturally built from simpler parts.
Exam Point — Key Theoretical Justifications for Depth

1. Circuit complexity: Certain functions require exponentially many neurons in 1 layer but only polynomially many in multiple layers.

2. Universal Approximation: Both shallow and deep networks are universal approximators, but depth improves parameter efficiency exponentially.

3. Compositionality argument: Real data is compositional; deep networks mirror this structure and can therefore learn it more efficiently.

Module 1 — Complete Summary

  • 1.
    A neural network is a composition of artificial neurons organized in layers. Each neuron computes z = W·x + b, then a = f(z).
  • 2.
    Forward propagation passes input through all layers to produce a prediction ŷ. The loss function measures how wrong ŷ is compared to y.
  • 3.
    Gradient Descent minimizes the cost by updating parameters in the direction opposite to the gradient: W := W − α·(∂J/∂W).
  • 4.
    Backpropagation efficiently computes all gradients using the chain rule in a single backward pass. It is not the optimizer — gradient descent is.
  • 5.
    Deep networks learn hierarchical, compositional representations. They are exponentially more parameter-efficient than shallow networks for complex functions.
  • 6.
    Vectorization replaces slow for-loops with matrix operations, enabling efficient parallel computation on CPUs and GPUs.
  • 7.
    Deep networks excel at structured data (images, audio, text) but require more data and careful training. Shallow networks suit simple, small-data problems.
