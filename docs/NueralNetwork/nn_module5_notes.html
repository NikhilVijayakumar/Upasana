Deep Learning in Computer Vision — Landscape

1.1 Why Deep Learning Transformed Computer Vision

Before deep learning, computer vision relied on hand-crafted features: SIFT, HOG, SURF, Gabor filters. Engineers spent years designing features that might capture edges, corners, and textures. Deep learning replaced all of this with learned features — the network discovers which features are useful directly from data.

  • ▸
    2012 turning point: AlexNet's 15.3% ImageNet top-5 error (vs ~26% for hand-crafted methods) demonstrated that CNNs could exceed human-engineered features by a massive margin. Every year since, deep models have improved further.
  • ▸
    Feature hierarchy: Deep CNNs automatically learn a hierarchy — edges → textures → parts → objects — that mirrors how neuroscientists believe the human visual cortex processes information (V1 → V2 → V4 → IT cortex).
  • ▸
    Transfer learning revolution: A CNN trained on ImageNet (1.2M images, 1000 classes) learns general visual features reusable across almost any vision task. Fine-tune the last few layers on your specific data — even with just hundreds of examples — and achieve strong performance.
  • ▸
    Scale: Modern vision models (ViT-22B, Florence) are trained on billions of image-text pairs and show emergent capabilities not present in smaller models.

Section 2

Object Detection — Architecture and Methods

2.1 The Object Detection Problem

Object detection requires answering two questions simultaneously for every object in the image: What is it? (classification) and Where is it? (localization via bounding box). Unlike classification, detection must handle a variable number of objects at varying scales and positions.

2.2 Two-Stage Detectors — R-CNN Family

R-CNN Pipeline (original)
Input Image
→
Selective Search (~2000 region proposals)
→
Warp each region to fixed size
→
CNN feature extraction (per region)
→
SVM classifier + Bounding box regressor
→
Detections
  • ▸
    R-CNN (2014): First deep learning detector. Extracts ~2000 region proposals with Selective Search, warps each to fixed size, runs CNN on each separately. Very slow: 47 seconds per image at test time.
  • ▸
    Fast R-CNN (2015): Key improvement: run CNN on the entire image ONCE to get a shared feature map. Then project each region proposal onto the feature map and use RoI (Region of Interest) Pooling to extract fixed-size features per region. 9× faster than R-CNN.
  • ▸
    Faster R-CNN (2015): Eliminates external Selective Search entirely. Adds a Region Proposal Network (RPN) — a small CNN that slides over the feature map and proposes candidate object regions (using anchor boxes of multiple scales and aspect ratios). End-to-end trainable. 140ms per image.
  • ▸
    Anchor boxes: Pre-defined bounding boxes of various scales (small, medium, large) and aspect ratios (1:1, 1:2, 2:1) centered at each position in the feature map. The RPN predicts offsets from these anchors rather than absolute coordinates.
  • ▸
    Mask R-CNN (2017): Extends Faster R-CNN by adding a mask prediction branch — for each detected object, predicts a pixel-wise binary mask. Enables instance segmentation (separate mask per object instance). RoIAlign (instead of RoIPool) improves spatial alignment.

2.3 One-Stage Detectors — YOLO Family

Two-stage detectors are accurate but slow. One-stage detectors predict bounding boxes and class probabilities in a single forward pass — making them suitable for real-time applications.

  YOLO (You Only Look Once) Core Idea:

  1. Divide image into S×S grid (e.g. 13×13)
  2. Each grid cell predicts:
     - B bounding boxes (with x, y, w, h, confidence)
     - C class probabilities
  3. Final detections = boxes × class probs, filtered by NMS

  Input (416×416)
       │
       ▼
  [Backbone CNN: DarkNet/CSPNet]  ←  feature extraction
       │
  [Neck: FPN / PANet]             ←  multi-scale feature fusion
       │
  [Detection Head]                ←  predict boxes at 3 scales
  ┌──────────┬──────────┬──────────┐
  │ 52×52    │ 26×26    │ 13×13   │  ← small, medium, large objects
  └──────────┴──────────┴──────────┘
  Post-processing: Non-Maximum Suppression (NMS) removes duplicate boxes
Figure 2.1 — YOLO architecture. Single forward pass, multi-scale detection heads.
  • ▸
    YOLOv1 (2015): First version. Treats detection as regression — directly predicts bounding boxes and class probabilities from the full image in one shot. Struggled with small objects and clusters.
  • ▸
    YOLOv3 (2018): Introduced multi-scale detection (3 scales at 3 different feature map resolutions). Much better at small objects. Still the most widely deployed version in industry.
  • ▸
    YOLOv8/v9 (2023–24): State-of-the-art speed-accuracy tradeoff. Anchor-free, decoupled head, task-aligned assignment. Runs in real-time on edge devices.
  • ▸
    Non-Maximum Suppression (NMS): Post-processing step. Multiple boxes may predict the same object. NMS keeps the highest-confidence box and suppresses overlapping boxes with IoU above a threshold. Essential for clean detections.
  • ▸
    IoU (Intersection over Union): Metric for bounding box overlap. IoU = Area(intersection) / Area(union). A predicted box with IoU > 0.5 with the ground truth box is typically considered a correct detection.

2.4 Detection Metrics — mAP

  • ▸
    Precision: Of all predicted detections, what fraction is correct? Precision = TP / (TP + FP).
  • ▸
    Recall: Of all actual objects, what fraction did we detect? Recall = TP / (TP + FN).
  • ▸
    Average Precision (AP): Area under the Precision-Recall curve for one class. Computed by sweeping the confidence threshold and plotting precision vs recall.
  • ▸
    mAP (mean AP): Mean of AP values across all C classes. The standard metric for comparing detectors on benchmarks like COCO and PASCAL VOC. Higher is better.
Exam Point — Two-Stage vs One-Stage Detectors

Two-stage (Faster R-CNN, Mask R-CNN): Stage 1 generates region proposals; Stage 2 classifies and refines them. Higher accuracy, especially for small objects. Slower (~5–10 FPS).

One-stage (YOLO, SSD): Single pass predicts all boxes. Faster (real-time, 30–100+ FPS). Slightly lower accuracy on dense small objects. Used in autonomous driving, surveillance, robotics.

Section 3

Image Segmentation — Semantic, Instance, Panoptic

3.1 Semantic Segmentation

Assign a class label to every single pixel in the image. Unlike detection (bounding boxes), segmentation provides exact object boundaries. All instances of the same class get the same label (cannot distinguish individual cars, only "car" vs "not car").

  • ▸
    Fully Convolutional Network (FCN): First deep learning segmentation model. Replaced FC layers with 1×1 Conv layers (making the classifier spatially aware). Added skip connections from earlier layers to recover spatial details lost during downsampling. Output: dense prediction of same H×W as input.
  • ▸
    U-Net (2015): Encoder-decoder with symmetric skip connections between corresponding encoder and decoder layers. Encoder captures context; decoder enables precise localization. Skip connections preserve fine-grained spatial detail. Dominant in medical image segmentation.
  • ▸
    DeepLab series: Uses Atrous (dilated) Convolutions — inserts holes (zeros) into the filter, expanding the receptive field without increasing parameters or reducing resolution. ASPP (Atrous Spatial Pyramid Pooling) captures context at multiple scales using different dilation rates in parallel.
  • ▸
    Dilated Conv receptive field: With dilation rate d, a 3×3 filter covers a (2d+1)×(2d+1) area. With d=6: covers 13×13. With d=12: covers 25×25. All without more parameters.

3.2 Instance Segmentation

Goes further than semantic segmentation — distinguishes individual object instances (Car #1 gets mask A, Car #2 gets mask B, even though both are "car"). Combines detection and segmentation.

  • ▸
    Mask R-CNN: Extends Faster R-CNN with a mask head — for each detected bounding box, predicts a 28×28 binary mask independently for each class. Three parallel outputs: class score + box offset + instance mask.
  • ▸
    RoIAlign: The key fix in Mask R-CNN over Faster R-CNN's RoIPool. RoIPool uses quantization (rounding) which misaligns features with spatial positions. RoIAlign uses bilinear interpolation to extract features at exact float coordinates — critical for pixel-accurate masks.

3.3 Medical Image Segmentation (Major Application)

  • ▸
    U-Net dominance: U-Net has become the standard backbone for virtually all medical image segmentation tasks: brain tumor (BraTS), organ segmentation (multi-organ challenge), cell segmentation (microscopy), retinal vessel segmentation.
  • ▸
    Challenges specific to medical imaging: Limited labeled data (expert annotation is expensive and time-consuming). Class imbalance (tumor pixels are tiny fraction of CT volume). 3D volumetric data (CT, MRI) requires 3D U-Net. Varying imaging protocols across hospitals.
  • ▸
    Dice Loss: Cross-entropy loss performs poorly with extreme class imbalance (e.g., tumor vs. background). Dice Loss directly optimizes the Dice coefficient (2|A∩B| / (|A|+|B|)) — a direct measure of segmentation overlap. Standard for medical segmentation.
  • ▸
    SAM (Segment Anything Model, Meta 2023): Foundation model for segmentation. Trained on 1 billion masks. Given any prompt (point, box, or text), segments any object in any image zero-shot. Transformative for medical imaging, satellite analysis, robotics.
Section 4

Other Major Computer Vision Applications

4.1 Image Captioning

  • ▸
    Task: Generate a natural language description of an image automatically.
  • ▸
    Architecture: CNN encoder (e.g. ResNet or ViT) extracts image features → fed as initial hidden state or context to an LSTM/Transformer decoder → decoder generates the caption word by word.
  • ▸
    Attention mechanism: The decoder attends to different spatial regions of the image at each word generation step. "Fluffy" attends to the texture; "dog" attends to the animal region. Makes captions more accurate and provides visual explanations.
  • ▸
    Applications: Accessibility (image descriptions for visually impaired), social media auto-captioning, content-based image retrieval, visual question answering.

4.2 Face Recognition and Verification

  • ▸
    Two tasks: Verification (are these two images the same person? 1:1 comparison) vs Recognition (who is this person? 1:N search in a database).
  • ▸
    Deep face embedding: A CNN maps each face image to a compact embedding vector (128–512 dimensional). Faces of the same person should be nearby in embedding space; different people should be far apart.
  • ▸
    Siamese Networks: Two identical CNNs (shared weights) process two face images and produce embeddings. A contrastive loss or cosine similarity measures distance between embeddings. Used for verification.
  • ▸
    Triplet Loss: Trained with triplets (anchor, positive, negative). Loss = max(0, d(anchor, positive) − d(anchor, negative) + margin). Forces the anchor to be closer to the same-person positive than to the different-person negative by at least a margin.
  • ▸
    Applications: Phone unlock (Face ID), airport boarding, law enforcement, access control. Ethical concerns: surveillance, bias across demographics.

4.3 Autonomous Driving Perception

  • ▸
    Sensor fusion: Autonomous vehicles use multiple sensor modalities: Camera (RGB), LiDAR (3D point cloud), RADAR (velocity), HD maps. Deep learning fuses these inputs for robust 3D perception.
  • ▸
    3D Object Detection: Detect pedestrians, vehicles, cyclists in 3D space (not just 2D bounding boxes). PointNet / PointPillar processes raw LiDAR point clouds.
  • ▸
    Lane detection: CNN-based segmentation models identify lane markings. Used for lane-keeping assist and highway autopilot.
  • ▸
    Optical flow: CNN predicts pixel-level motion between consecutive video frames. Used for understanding scene dynamics, predicting future states.

4.4 Medical Imaging Applications

Radiology / Pathology

CNN detects pneumonia in chest X-rays (CheXNet). Classifies diabetic retinopathy from retinal fundus images. Detects cancer from histopathology slides (whole-slide images).

Dermatology

Classify skin lesions (benign vs malignant melanoma) from dermoscopy images. Stanford's DermNet CNN matched board-certified dermatologist performance.

Radiology AI

Auto-detect COVID-19 from CT scans. Measure organ volumes from MRI. Detect vertebral fractures, pulmonary nodules. Reduces radiologist workload.

Drug Discovery

CNN analyzes microscopy images of cells after drug treatment. Identifies drug candidates based on cell morphology changes. High-throughput screening at scale.

Deep Learning in NLP — The Revolution

5.1 Why Deep Learning Transformed NLP

Before deep learning, NLP relied on hand-engineered features (n-gram counts, TF-IDF), sparse representations (bag of words), and shallow models (Naive Bayes, SVM, logistic regression). These methods could not capture semantics, long-range dependencies, or the ambiguity of natural language.

  • ▸
    Distributed representations: Deep learning represents words as dense, low-dimensional vectors (embeddings) where semantically similar words are nearby. "King" and "Queen" are close; "King" and "Carrot" are far. This captures meaning that sparse BOW cannot.
  • ▸
    Context sensitivity: The word "bank" means different things in "river bank" and "bank account." Deep models (especially Transformers) produce context-dependent representations — the embedding of "bank" changes based on surrounding words.
  • ▸
    Long-range dependencies: In "The cat, which sat on the mat that was in the living room of the old house, was hungry", "cat" and "was hungry" are connected across 20 words. RNNs struggle; Transformers handle this naturally via self-attention.
  • ▸
    Pre-training paradigm: Train on massive unlabeled text (self-supervised) → fine-tune on small labeled datasets. GPT-3 was trained on 500B tokens; GPT-4 on trillions. This massively reduces data requirements for downstream tasks.

Section 6

Word Embeddings — Representing Language

6.1 One-Hot Encoding (Baseline)

  • ▸
    Representation: Vocabulary of V words → each word is a V-dimensional vector with a single 1 and V-1 zeros.
  • ▸
    Problems: High dimensionality (V can be 100,000+). No notion of similarity — cosine distance between any two distinct words is 1.0. "Cat" and "Kitten" are as similar as "Cat" and "Submarine."

6.2 Word2Vec (Mikolov et al., 2013)

Train a shallow neural network to predict either (a) the context from the word, or (b) the word from its context. The hidden layer weights become the word embeddings.

  • ▸
    CBOW (Continuous Bag of Words): Given surrounding context words, predict the center word. Input: context word embeddings → average → predict center word via softmax. Fast to train, works well for frequent words.
  • ▸
    Skip-gram: Given one center word, predict surrounding context words. Input: center word → predict each context word. Slower but better for rare words and analogies.
  • ▸
    Semantic arithmetic: Learned embeddings capture semantic relationships: vec("King") − vec("Man") + vec("Woman") ≈ vec("Queen"). vec("Paris") − vec("France") + vec("Italy") ≈ vec("Rome"). Remarkable emergent property.
  • ▸
    Limitation: Context-independent — "bank" always has the same embedding regardless of context. Pre-trained embeddings are static.

6.3 GloVe (Global Vectors for Word Representation)

  • ▸
    Idea: Word2Vec uses local context windows. GloVe uses global word-word co-occurrence statistics across the entire corpus. Trains embeddings to satisfy: wᵢᵀ·w̃ⱼ + bᵢ + b̃ⱼ = log Xᵢⱼ, where Xᵢⱼ is the co-occurrence count.
  • ▸
    Advantage: Combines the strengths of global matrix factorization (LSA) and local context window methods (Word2Vec). Often produces better analogical reasoning.

6.4 Contextualized Embeddings — ELMo

  • ▸
    Problem with Word2Vec/GloVe: Same embedding for "play" whether it's "play guitar" or "a play at the theatre." Language is deeply context-dependent.
  • ▸
    ELMo (Embeddings from Language Models): Uses a bidirectional LSTM language model. The embedding for each word is derived from all internal LSTM states of the entire sentence — different contexts give different embeddings. "Bank" has a different representation in each sentence.
  • ▸
    Significance: ELMo demonstrated the power of contextual representations, which was then taken much further by BERT and GPT using Transformers.
Section 7

Transformers and Attention in NLP

7.1 The Transformer Architecture

The Transformer (Vaswani et al., "Attention Is All You Need," 2017) replaced RNNs in NLP entirely for most tasks. It processes sequences in parallel (not sequentially) using Self-Attention mechanisms.

  TRANSFORMER ENCODER BLOCK:

  Input tokens: [x₁, x₂, ..., xₙ]
       │
  [Token Embedding + Positional Encoding]
       │
  ┌────────────────────────────────────┐
  │  Multi-Head Self-Attention          │  ← attends to ALL other tokens
  │  Add & Norm (Residual connection)  │
  │  Feed-Forward Network (per token)  │  ← independent dense layers
  │  Add & Norm                        │
  └────────────────────────────────────┘  ← stack N times (e.g. 12 layers)
       │
  [CLS] representation → Classification head
  All token representations → Sequence tasks (NER, QA)
Figure 7.1 — Transformer Encoder block. Self-attention + FFN + residual connections, stacked N times.

7.2 Self-Attention — The Core Mechanism

Scaled Dot-Product Self-Attention
For each token, compute Query (Q), Key (K), Value (V):
Q = X · Wq, K = X · Wk, V = X · Wv ← learned projections

Attention(Q, K, V) = softmax( Q·Kᵀ / √dₖ ) · V

Q·Kᵀ: (n×dₖ) · (dₖ×n) = (n×n) ← attention score matrix
/ √dₖ: scale to prevent softmax saturation
softmax: normalize scores to probabilities → how much to attend to each token
×V: weighted sum of value vectors → new contextual representation
  • ▸
    Query-Key-Value intuition: Q = "what am I looking for?", K = "what do I contain?", V = "what information will I contribute?". Attention score between tokens i and j = how much Q of token i matches K of token j.
  • ▸
    O(n²) attention: Every token attends to every other token. Attention score matrix is n×n. For a sentence of 512 tokens: 512² = 262,144 scores computed. This is the main computational bottleneck for long sequences.
  • ▸
    Multi-Head Attention: Run h parallel attention heads with different Wq, Wk, Wv projections. Each head learns to attend to different aspects (syntax, semantics, coreference). Concatenate all heads, project back to model dimension.
  • ▸
    Positional Encoding: Unlike RNNs, Transformers have no inherent notion of sequence order. Positional encodings (sinusoidal or learned) are added to token embeddings to provide position information.

7.3 BERT — Bidirectional Encoder

  • ▸
    Architecture: 12 Transformer encoder blocks (BERT-base) or 24 (BERT-large). Processes the entire sequence bidirectionally — each token attends to ALL other tokens in both directions simultaneously.
  • ▸
    Pre-training tasks: (1) Masked Language Modeling (MLM): randomly mask 15% of tokens, predict them from context. (2) Next Sentence Prediction (NSP): predict whether two sentences are consecutive. Trained on 3.3B words (BooksCorpus + Wikipedia).
  • ▸
    Fine-tuning: Add a task-specific head (classification, NER, QA span predictor) on top of BERT and fine-tune all weights on labeled task data. BERT set new state-of-the-art on 11 NLP benchmarks simultaneously on release.
  • ▸
    [CLS] token: A special token prepended to every input. Its final representation serves as the aggregate sequence representation for classification tasks.

7.4 GPT Family — Autoregressive Decoder

  • ▸
    Architecture: Transformer decoder blocks. Uses causal (masked) self-attention — each token only attends to previous tokens (left-to-right). Pre-trained on next-word prediction (autoregressive language modeling).
  • ▸
    GPT-3 (2020): 175 billion parameters. Demonstrated few-shot and zero-shot learning — given a few examples in the prompt, it can perform new tasks without any gradient updates (in-context learning).
  • ▸
    ChatGPT / GPT-4: GPT + Reinforcement Learning from Human Feedback (RLHF). Human raters rank model outputs; a reward model is trained on these rankings; GPT is fine-tuned with PPO to maximize reward. Makes the model helpful, harmless, and honest.
  • ▸
    BERT vs GPT: BERT: bidirectional, best for understanding tasks (classification, NER, QA). GPT: unidirectional, best for generation tasks (text generation, dialogue, code). Modern LLMs (GPT-4, Claude, Gemini) are GPT-style but with many improvements.
Section 8

NLP Applications — In Depth

8.1 Machine Translation

  • ▸
    Evolution: Rule-based → Statistical MT (phrase tables) → Neural MT (Seq2Seq + Attention) → Transformer MT. Each step dramatically improved fluency and accuracy.
  • ▸
    Transformer MT: Encoder processes source sentence; decoder generates target sentence autoregressively. Attention between encoder and decoder (cross-attention) aligns source and target positions.
  • ▸
    BLEU score: Bilingual Evaluation Understudy. Measures n-gram overlap between generated translation and reference translations. Standard evaluation metric. Higher is better (0–100 scale).
  • ▸
    Real-world deployment: Google Translate (100+ languages), DeepL, AWS Translate all use Transformer-based neural MT. Google processes billions of translation requests daily.

8.2 Sentiment Analysis

  • ▸
    Task: Determine opinion polarity (positive/negative/neutral) or fine-grained sentiment from text.
  • ▸
    Architectures used: Early: LSTM/BiLSTM → Transformer (BERT fine-tuned on SST, IMDB). BERT achieves 95%+ accuracy on standard benchmarks.
  • ▸
    Aspect-based SA: Determine sentiment toward specific aspects: "The food was great but the service was terrible." → food: positive, service: negative.
  • ▸
    Applications: Brand monitoring, customer feedback analysis, political opinion tracking, financial market sentiment.

8.3 Question Answering (QA)

  • ▸
    Extractive QA: Given a document and question, identify the exact span of text in the document that answers the question. BERT fine-tuned on SQuAD dataset — predicts start and end token positions of the answer span.
  • ▸
    Open-domain QA: No document given — the model must retrieve relevant documents first, then extract/generate the answer. Retrieval-Augmented Generation (RAG) — neural retriever finds relevant passages; generator produces answer.
  • ▸
    Generative QA: Generate a free-form answer (not just extract a span). GPT-style models excel here. Evaluated by human judges or LLM-as-judge on criteria like correctness, fluency, faithfulness.

8.4 Text Summarization

  • ▸
    Extractive: Select the most important sentences from the source text. No generation — just selection. BertSum: BERT encodes the document, a classifier selects sentences.
  • ▸
    Abstractive: Generate a new, shorter text that captures the main points — may use words not in the original. Seq2Seq Transformer (BART, T5). Can produce more fluent summaries than extractive.
  • ▸
    ROUGE metric: Recall-Oriented Understudy for Gisting Evaluation. Measures n-gram recall between generated summary and reference summaries. Standard evaluation metric alongside human evaluation.

8.5 Named Entity Recognition (NER)

  • ▸
    Task: Label each token as a named entity type: Person (PER), Organization (ORG), Location (LOC), Date, etc., or Other (O).
  • ▸
    Architecture: BiLSTM + CRF was the standard for years. Now: BERT fine-tuned with a token classification head. CRF (Conditional Random Field) layer on top enforces label dependencies (e.g., I-PER cannot follow B-ORG).
  • ▸
    Applications: Information extraction, knowledge graph construction, medical record parsing (diseases, drugs, symptoms), legal document analysis.

Deep Learning for Wind Estimation

9.1 Why Wind Estimation Matters

Accurate wind estimation and forecasting is critical across many high-stakes applications. Errors in wind prediction directly cost lives (aviation safety), billions in revenue (energy trading), and impact climate models. Traditional numerical methods (NWP — Numerical Weather Prediction) are computationally expensive and struggle with local terrain effects and rapidly changing conditions.

Wind Energy

Wind farms must predict output hours/days ahead for grid integration and energy trading. 1% improvement in wind forecast accuracy = millions in revenue gains for large wind farms.

Aviation Safety

Low-level wind shear, turbulence, and wake turbulence are major hazards. Deep learning improves wind field estimation from radar, lidar, and ATIS data for safer approach/departure.

Structural Engineering

Bridges, skyscrapers, and offshore platforms must account for wind loads. Real-time deep learning models estimate wind pressure distributions for structural health monitoring.

Climate Science

Accurate regional wind patterns are essential for climate modeling, pollutant dispersion modeling, wildfire spread prediction, and agricultural planning.

9.2 Wind Data Sources for Deep Learning

  • ▸
    Numerical Weather Prediction (NWP) outputs: ERA5, GFS, ECMWF reanalysis datasets. High spatial and temporal resolution gridded wind field data. Used both as input features and as training targets.
  • ▸
    Anemometer measurements: Point measurements from met masts and weather stations. High temporal resolution (1Hz–1min). Spatially sparse — only at instrumented locations.
  • ▸
    LiDAR (Light Detection And Ranging): Remote sensing technology that scans a wind field volume using laser pulses. Provides detailed 2D/3D wind profiles. Used in wind farm assessment, aviation.
  • ▸
    SCADA data: Supervisory Control and Data Acquisition — wind turbine operational data (rotor speed, power output, pitch angle) used to infer wind conditions from turbine response.
  • ▸
    Satellite imagery: Sea surface roughness from SAR (Synthetic Aperture Radar) correlates with near-surface wind speed over oceans. Deep learning extracts wind from SAR images.

9.3 Deep Learning Approaches for Wind Forecasting

A. Short-Term Wind Speed Forecasting (Seconds to Hours)

  • ▸
    LSTM/GRU-based: Use historical time series of wind speed, direction, and meteorological variables to forecast future wind speed at a single point. LSTM captures temporal dependencies in the wind time series.
  • ▸
    CNN-LSTM hybrid: CNN extracts spatial patterns from gridded NWP data or wind field images; LSTM models temporal evolution. Particularly effective for turbine wake modeling in wind farms.
  • ▸
    Encoder-Decoder with Attention: Attends to the most relevant historical time steps when making each future prediction. Provides interpretability — which past conditions drove the forecast?

B. Wind Field Reconstruction and Super-Resolution

  • ▸
    Problem: NWP models have coarse spatial resolution (several km). Wind farms and urban applications need fine-grained wind fields (tens of meters). Downscaling = spatial super-resolution of wind.
  • ▸
    CNN / U-Net for super-resolution: Train a CNN to map coarse NWP wind fields to high-resolution fields (using simulation data as ground truth). Deep learning surpasses traditional statistical downscaling methods significantly.
  • ▸
    GAN for wind super-resolution: SRGAN-style architecture applied to wind fields. Generator creates high-resolution wind field; discriminator ensures it looks physically realistic. Captures fine-grained turbulence that MSE-trained networks miss.

C. Wake Effect Modeling in Wind Farms

  • ▸
    Wake effect: Upstream turbines extract energy from wind and create a wake (region of reduced speed, increased turbulence) that impacts downstream turbines. Accurate wake modeling is critical for farm layout optimization and power prediction.
  • ▸
    Physics-Informed Neural Networks (PINNs): Embed the Navier-Stokes equations as a loss term. Network learns to satisfy both training data AND physical fluid dynamics laws. Enables wake field reconstruction from sparse sensor measurements.
  • ▸
    Graph Neural Networks: Model the wind farm as a graph — turbines are nodes, aerodynamic interactions are edges. GNN propagates information across the graph to predict power output and wake patterns for each turbine.

D. Extreme Wind Event Detection

  • ▸
    Task: Early detection of hurricanes, typhoons, thunderstorm downbursts, and other extreme wind events from satellite imagery and NWP data.
  • ▸

  • ▸
    GraphCast (DeepMind, 2022): Graph Neural Network trained on ERA5 reanalysis data. Predicts global weather (including wind at 37 pressure levels) at 10-day forecast horizon in under 1 minute — vs 1 hour for ECMWF NWP. Outperforms NWP on most variables.

Exam Point — Why Deep Learning Outperforms Traditional Methods for Wind

1. Non-linear patterns: Wind turbulence, terrain effects, and mesoscale interactions are highly non-linear — beyond the capacity of linear regression or ARIMA models.

2. Multi-source fusion: Deep learning naturally integrates heterogeneous data sources (NWP, LiDAR, SCADA) into a unified prediction framework.

3. Spatial-temporal learning: CNN+LSTM or Transformer architectures jointly capture spatial wind field patterns AND temporal evolution — impossible with classical statistical models.

4. Physics integration: PINNs incorporate fluid dynamics constraints, ensuring predictions are physically consistent even where training data is sparse.

Section 11

Explainability, Interpretability and Fairness

11.1 The Black Box Problem

Deep neural networks are fundamentally opaque — given an input, they produce an output, but the internal reasoning is inscrutable. This is a critical barrier to deployment in high-stakes domains (medicine, law, finance, autonomous systems) where decisions must be explainable and auditable.

Explainability Method
LIME — Local Interpretable Model-Agnostic Explanations

Perturbs the input (mask parts of image/text), observes how the output changes, fits a simple linear model locally to approximate the deep model's behavior near that input. Works with any black-box model.

Output: which input features most influenced this prediction.

Applications: medical diagnosis, loan approval, fraud detection
Explainability Method
SHAP — SHapley Additive exPlanations

Based on cooperative game theory (Shapley values). Assigns each feature a contribution score — how much did this feature shift the model's prediction from the average? Consistent and theoretically grounded.

Unified framework: works for tabular, image, and text models.

Widely used in finance, healthcare, regulatory compliance (GDPR)
Explainability Method
Grad-CAM — Gradient-weighted Class Activation Maps

Uses gradients of the target class score flowing into the final Conv layer to produce a coarse localization map showing which regions of the image were most important for the classification decision.

Requires no architectural change — works on any CNN.

Medical imaging, autonomous driving, content moderation
Fairness Research
Algorithmic Bias and Fairness

Deep learning models trained on biased historical data replicate and amplify those biases. Face recognition systems perform worse on darker skin tones. Hiring algorithms disadvantage women. Medical models perform worse on underrepresented groups.

Active research: measuring bias (demographic parity, equalized odds), debiasing training data, adversarial debiasing, fairness-aware loss functions.

Regulation: EU AI Act, US EEOC guidelines require fairness audits

11.2 Adversarial Robustness

  • ▸
    Adversarial examples: Tiny, imperceptible perturbations added to an input (e.g., an image) can cause a deep network to misclassify with high confidence. A panda image with invisible noise → classified as "gibbon" by ResNet with 99.3% confidence. Humans notice nothing.
  • ▸
    Why it matters: Self-driving cars, medical AI, and security systems could be fooled by adversarial attacks in the real world. A stop sign with an adversarial patch → misclassified as "speed limit."
  • ▸
    Adversarial training: Include adversarially perturbed examples during training. Improves robustness but reduces clean accuracy — a fundamental tradeoff. An active research area with no fully satisfactory solution yet.
  • ▸
    Certified robustness: Prove mathematically that within a ball of radius ε around any input, the model's prediction is constant. Randomized smoothing, interval bound propagation — computationally expensive but providing guarantees.
Section 12

Efficient Deep Learning and Federated Learning

12.1 Efficient Deep Learning

Modern models (GPT-4, Gemini Ultra, PaLM) have trillions of parameters and require enormous compute for training and inference. Deploying these on edge devices (smartphones, IoT, embedded systems) is impossible without model compression.

Compression Technique
Model Pruning

Remove neurons, connections, or entire layers that contribute least to model performance (measured by weight magnitude or gradient sensitivity). After pruning, retrain to recover accuracy. Can achieve 10–100× compression with minimal accuracy loss.

Unstructured pruning: remove individual weights → sparse matrices. Structured pruning: remove entire filters/channels → smaller dense network. Structured is more hardware-friendly.

Lottery Ticket Hypothesis: a dense network contains a small sparse sub-network that can match full performance
Compression Technique
Knowledge Distillation

Train a small "student" model to mimic the outputs of a large pre-trained "teacher" model. Student learns from the teacher's soft probability outputs (not just hard labels) — these contain richer information about inter-class similarities.

Loss = α · cross-entropy(student, labels) + (1−α) · KL(student, teacher_softmax). The teacher's soft outputs are "dark knowledge."

DistilBERT: 40% smaller, 60% faster than BERT, retains 97% of BERT's language understanding
Compression Technique
Quantization

Reduce numerical precision of weights and activations. Standard: FP32 (32-bit float). Quantized: INT8 (8-bit int), INT4, or even 1-bit (binary). INT8 quantization: 4× memory reduction, 2–4× inference speedup on compatible hardware with ~1% accuracy loss.

Post-training quantization: quantize after training. Quantization-aware training: simulate quantization during training → better accuracy.

Essential for mobile deployment: CoreML, TFLite, ONNX Runtime
Architecture Design
Neural Architecture Search (NAS)

Automate the design of neural network architectures. Search over a space of possible architectures to find the optimal one for a given task and hardware constraint. Methods: reinforcement learning (controller proposes architectures), evolutionary algorithms, differentiable NAS (DARTS).

EfficientNet was discovered via NAS. MobileNetV3 also uses NAS. Dramatic improvements over hand-designed architectures.

EfficientNet: 8× smaller + 6× faster than ResNet with better accuracy

12.2 Federated Learning

  • ▸
    The privacy problem: Training deep learning models requires large amounts of data. In many domains (healthcare, finance, mobile), this data is sensitive and cannot be moved to a central server due to privacy regulations (GDPR, HIPAA).
  • ▸
    Federated Learning (FL) solution: Bring the model to the data, not the data to the model. Each device (hospital, phone, bank) trains the model locally on its private data. Only model updates (gradients or weights) are sent to the central server — never raw data.
  • ▸
    FedAvg algorithm: Server sends global model to all clients. Each client trains locally for E epochs. Clients send back updated weights. Server averages all client weights (weighted by dataset size) to get new global model. Repeat.
  • ▸
    Challenges: Non-IID data (each client's data distribution is different — e.g., each hospital has different patient demographics). Heterogeneous hardware (different clients have different compute). Communication efficiency (sending large weight updates is expensive). Privacy attacks (gradient inversion can recover training data).
  • ▸
    Applications: Google's Gboard keyboard trains next-word prediction on your phone without uploading your typing. Hospital consortiums train disease detection models without sharing patient records.
Exam Point — Federated Learning in 3 Points

1. Purpose: Enable training on distributed, private data without centralization — privacy-preserving deep learning.

2. Key mechanism: Local training + weight averaging (FedAvg). Only gradients/weights leave the device, never raw data.

3. Main challenges: Non-IID data distribution, communication cost, and gradient inversion attacks that may still leak private information.

Section 13

Self-Supervised Learning and Foundation Models

13.1 Self-Supervised Learning (SSL)

The dominant paradigm in modern deep learning. The core idea: create supervised learning signals automatically from unlabeled data by defining a pretext task, eliminating the need for expensive manual annotation.

SSL Method
Masked Autoencoders (MAE)

Randomly mask 75% of image patches. Train encoder-decoder to reconstruct the masked patches. The encoder must learn rich semantic features to reconstruct from only 25% visible information. ViT pre-trained with MAE achieves SOTA on ImageNet.

He et al. (2021). Generalization: BEiT (BERT for images), VideoMAE (video)
SSL Method
Contrastive Learning (SimCLR, MoCo)

Create two augmented views of the same image (positive pair). Train encoder so same-image representations are close (attract) and different-image representations are far apart (repel). No labels needed. Representations rival supervised pre-training.

Loss: InfoNCE = −log[ exp(sim(z₁,z₂)/τ) / Σexp(sim(z₁,zₖ)/τ) ]

DINO, MoCo v3, SimCLR — used widely for vision pre-training
SSL Method
Language Model Pre-training

Next token prediction (GPT family) and Masked Token Prediction (BERT family) are both SSL tasks — the label is created from the data itself. Training on trillions of tokens builds rich world knowledge and reasoning ability without any human annotation.

GPT-4, Claude, Gemini, LLaMA, Mistral — all SSL pre-trained

13.2 Foundation Models

  • ▸
    Definition: Large models trained on broad data at scale that can be adapted (fine-tuned, prompted) for a wide variety of downstream tasks. The term was coined by Stanford HAI (2021). They are called "foundation" because other models and applications are built on top of them.
  • ▸
    Key properties: Emergent abilities — capabilities not present in smaller models that appear only at scale (few-shot reasoning, chain-of-thought, arithmetic). Transfer to many tasks with minimal or zero task-specific training data.
  • ▸
    Examples: Language: GPT-4, Claude 3, Gemini Ultra, LLaMA 3. Vision: ViT-22B, SAM, DINOv2. Multimodal: GPT-4V, LLaVA, CLIP, Flamingo. Science: AlphaFold2 (protein structure), ESM-2 (protein language model), GraphCast (weather).
  • ▸
    Instruction tuning: Fine-tune pre-trained LLMs on examples of instruction-following (Q&A, task descriptions). Dramatically improves usability and alignment without needing task-specific datasets.
  • ▸
    RLHF (Reinforcement Learning from Human Feedback): Human raters rank model responses → train reward model → fine-tune LLM with PPO to maximize reward. Used in ChatGPT, Claude, Gemini — aligns models with human preferences and values.

13.3 In-Context Learning and Prompting

  • ▸
    Zero-shot: Describe the task in natural language, no examples. GPT-4: "Translate to French: 'Hello world'" — no training needed.
  • ▸
    Few-shot: Provide a few examples in the prompt, then the query. Model infers the pattern and applies it — no gradient update. Emergent in models above ~10B parameters.
  • ▸
    Chain-of-Thought (CoT): Include step-by-step reasoning examples in the prompt. Model produces intermediate reasoning steps before the final answer. Dramatically improves multi-step math, logical, and scientific reasoning.
Section 14

Multimodal Models, Physics-Informed Learning and Neuro-AI

14.1 Multimodal Deep Learning

Active Research Area
Vision-Language Models

Models that jointly process images and text. CLIP (Contrastive Language-Image Pre-training) trains CNN image encoder + Transformer text encoder together on 400M image-text pairs with contrastive loss — aligns image and text representations in a shared embedding space.

Enables zero-shot image classification: compute similarity of image to text prompts "a photo of a dog", "a photo of a cat" → label = highest similarity.

DALL-E, Stable Diffusion, LLaVA, GPT-4V, Flamingo, CogVLM
Active Research Area
Text-to-Image Generation

Generate photorealistic images from text descriptions. Diffusion models (DDPM, DDIM) iteratively denoise random noise guided by a text embedding (CLIP). Stable Diffusion operates in latent space for efficiency.

Key components: CLIP text encoder → U-Net denoiser → VAE decoder. Score function guides the denoising process toward the text description.

DALL-E 3, Midjourney, Stable Diffusion, Imagen, Firefly
Active Research Area
Diffusion Models

State-of-the-art generative models that have largely superseded GANs for image/audio synthesis. Forward process: gradually add Gaussian noise to data over T steps. Reverse process: learn to denoise step by step. Samples generated by running the learned reverse process from pure noise.

Advantages over GANs: stable training, no mode collapse, better sample diversity and quality on complex distributions.

Sora (video), AudioLM (audio), RFdiffusion (protein design)

14.2 Physics-Informed Neural Networks (PINNs)

  • ▸
    Core idea: Embed physical laws (PDEs, ODEs, conservation laws) directly into the neural network loss function. The network must simultaneously fit training data AND satisfy the governing physics equations at collocation points.
  • ▸
    Loss = Data loss + Physics loss: L = λ_data · ‖u_pred − u_data‖² + λ_physics · ‖N[u_pred]‖² where N is the PDE residual (how much the network output violates the differential equation).
  • ▸
    Advantages: Works with very sparse data (physics fills in the gaps). Predictions are physically consistent. No need to discretize the domain (mesh-free). Naturally enforces conservation laws.
  • ▸
    Applications: Fluid mechanics (Navier-Stokes), heat transfer, structural mechanics, electromagnetism, quantum mechanics, epidemiological modeling, wind farm simulation.

14.3 Neuro-Inspired AI and Continual Learning

Research Direction
Continual / Lifelong Learning

Standard deep networks suffer from catastrophic forgetting — learning a new task destroys performance on old tasks because weight updates for the new task overwrite old task knowledge.

Research: Elastic Weight Consolidation (EWC) — regularize updates to important weights. Progressive Neural Networks — add new capacity for new tasks. Replay methods — rehearse old task examples while learning new ones.

Critical for robotics, personalized AI, autonomous agents
Research Direction
Spiking Neural Networks (SNNs)

Third generation of neural networks inspired by biological neuron firing. Neurons communicate via discrete spikes (binary events) rather than continuous activations. Sparse, event-driven computation — far more energy-efficient.

Challenge: not differentiable → cannot use standard backprop. Research: surrogate gradients, spike timing dependent plasticity (STDP). Deployed on neuromorphic chips (Intel Loihi, IBM TrueNorth).

Edge AI, brain-computer interfaces, ultra-low-power IoT
Research Direction
Graph Neural Networks (GNNs)

Extend deep learning to graph-structured data where relationships between entities are as important as the entities themselves. Message passing: each node aggregates information from its neighbors iteratively to update its representation.

Types: GCN (Graph Convolutional), GAT (Graph Attention), GraphSAGE, GIN. Deep learning now handles molecules, social networks, knowledge graphs, physical simulations natively.

AlphaFold2 (protein folding), drug discovery, recommendation systems
Research Direction
Causal Deep Learning

Standard deep learning learns correlations, not causation. Spurious correlations in training data lead to failures on distribution shift. A model may learn "grass is associated with cows" instead of learning what cows actually look like.

Research: Invariant Risk Minimization (IRM), causal representation learning, counterfactual reasoning. Goal: models that understand cause-effect relationships and generalize to new environments.

Medical AI, scientific discovery, policy evaluation
Research Direction
AI Safety and Alignment

As AI systems become more powerful, ensuring they behave safely, reliably, and in accordance with human values becomes critical. Key problems: reward hacking, distributional shift, scalable oversight, deceptive alignment.

Research: Constitutional AI (Anthropic), RLHF, interpretability (mechanistic interpretability — reverse-engineer what circuits in a neural network compute), debate, amplification.

Directly relevant to deployment of LLMs and autonomous AI systems
Research Direction
AI for Scientific Discovery

Deep learning as a tool for accelerating science. AlphaFold2 solved protein structure prediction (50-year grand challenge) with near-experimental accuracy. Similar breakthroughs are being sought in materials science, drug discovery, mathematics, climate science, and particle physics.

Key tools: GNNs for molecular property prediction, Transformers for genomics, diffusion models for molecular design, PINNs for physics simulation.

AlphaFold2 (proteins), MusicLM (music), AlphaGeometry (math proofs)
