Section 1

What is Reinforcement Learning?

1.1 The Learning Paradigm

Reinforcement Learning (RL) is a third paradigm of machine learning, distinct from supervised and unsupervised learning. An agent learns to make decisions by interacting with an environment, receiving feedback in the form of rewards or penalties. The goal is to learn a policy — a strategy for choosing actions — that maximizes cumulative reward over time.

The RL Problem in One Sentence

An agent at each timestep observes the current state, takes an action, receives a reward, transitions to a new state, and updates its behavior to maximize the total future reward — learning entirely from experience, with no labeled examples of correct actions provided.

1.2 RL vs Supervised vs Unsupervised Learning

AspectSupervised LearningUnsupervised LearningReinforcement Learning

Feedback type Immediate, complete (correct answer known) None Delayed, sparse, evaluative (good/bad, not correct action)
Data generation Fixed dataset, i.i.d. Fixed dataset Agent generates its own data through interaction
Key challenge Generalization Representation Credit assignment, exploration vs exploitation
Example Image classification Clustering, PCA Playing chess, robot navigation

  │  ┌──────────┐    action aₜ      ┌──────────────────────┐     │
  │  │          │──────────────────▶│                      │     │
  │  │  AGENT   │                   │    ENVIRONMENT       │     │

  At each timestep t:

  2. Agent selects action aₜ according to policy π
  3. Environment transitions to new state sₜ₊₁

  6. Repeat until terminal state or time limit

1.4 Key Characteristics of RL

  • ▸
    Trial and error learning: No teacher provides correct actions. The agent must discover which actions lead to high reward by trying them. This requires exploration — the agent cannot just exploit what it already knows.
  • ▸
    Delayed rewards: A reward may be received long after the action that caused it. In chess, a bad move in the opening may only reveal its consequences 30 moves later. The agent must solve the credit assignment problem — attributing outcomes to the actions that caused them.
  • ▸
    Sequential decision making: Actions affect not just immediate reward but also future states and future rewards. The agent must consider long-term consequences, not just immediate payoffs.
  • ▸
    Non-stationary data: As the agent's policy changes, the distribution of states it visits changes too. The data generation process is non-stationary — unlike supervised learning where the dataset is fixed.
Section 2

RL Components and Taxonomy

2.1 Core Components

  • ▸
    Agent: The decision-making entity. Perceives the environment's state, chooses actions, and updates its knowledge from experience. In RL, the agent can be a robot, a game-playing program, a trading system, or any autonomous decision-maker.
  • ▸
    Environment: Everything outside the agent. The agent interacts with it through actions, and it responds by providing new states and rewards. Can be deterministic (same action always leads to same next state) or stochastic (outcomes have randomness).
  • ▸
    State (s ∈ S): A description of the environment at a given time, containing all information relevant to future decisions. Can be fully observable (agent sees everything) or partially observable (agent sees a noisy/partial view).
  • ▸
    Action (a ∈ A): A choice the agent makes. Can be discrete (finite number of choices: up/down/left/right) or continuous (a real-valued vector: steering angle, force). The action space A can be state-dependent.
  • ▸
    Reward (r ∈ ℝ): A scalar feedback signal from the environment. Encodes the objective: what the agent should achieve. Reward design is critical and non-trivial. Sparse rewards (only at the end) are harder to learn from than dense rewards (at every step).
  • ▸
    Policy (π): The agent's strategy — a mapping from states to actions. Deterministic policy: π(s) = a (always takes action a in state s). Stochastic policy: π(a|s) = P(action = a | state = s). The ultimate goal is to find the optimal policy π*.
  • ▸
    Value function (V, Q): Estimates how good a state (V) or state-action pair (Q) is in terms of expected future reward. Used by the agent to evaluate and improve its policy without explicitly knowing the environment's dynamics.
  • ▸
    Model (optional): The agent's internal model of the environment — predicts next state and reward given current state and action. Model-based RL uses this for planning. Model-free RL (Q-Learning, Policy Gradient) learns directly from experience without a model.

2.3 Return — The Objective

The agent's objective is to maximize the expected return — the total accumulated reward over time. Simply summing all future rewards is problematic because distant rewards are less certain. We use discounted return:

Discounted Return
Gₜ = rₜ₊₁ + γ·rₜ₊₂ + γ²·rₜ₊₃ + ... = Σₖ₌₀^∞ γᵏ · rₜ₊ₖ₊₁

Gₜ = return from time step t onwards
rₜ₊ₖ = reward received k steps after t
γ ∈ [0,1) = discount factor

γ ≈ 0: Myopic — only immediate rewards matter. Short-sighted.
γ ≈ 1: Far-sighted — future rewards matter almost as much as immediate.
γ = 0.99 is typical in practice.

Why discount? 1. Mathematical convergence (bounded sum for γ<1)
2. Uncertainty increases further into the future
3. Matches economic intuition (present value)
  • ▸
    Recursive relation: Gₜ = rₜ₊₁ + γ · Gₜ₊₁. The return at time t equals immediate reward plus discounted return from next step. This recursive structure is the foundation of Bellman equations.
  • ▸
    Episode vs Continuing tasks: Episodic tasks have a natural end (chess game ends, maze is solved). Continuing tasks have no end (stock trading). For continuing tasks, discounting ensures Gₜ is finite.

Markov Decision Process — Formal Definition

3.1 The Markov Property

The foundation of all MDP-based RL is the Markov Property: the future depends only on the current state, not on the history of how we got there.

Markov Property (Memoryless Property)

P(sₜ₊₁ | sₜ, aₜ, sₜ₋₁, aₜ₋₁, ..., s₀, a₀) = P(sₜ₊₁ | sₜ, aₜ)

The transition probability to the next state depends only on the current state and action — not on any previous states or actions. The current state contains all relevant history.

Intuition: In chess, knowing the current board position is sufficient to play optimally — you don't need to know how the position was reached. The entire history is encoded in the current state.

3.2 Formal MDP Definition

A Markov Decision Process is defined by a 5-tuple: (S, A, P, R, γ)

  • S
    State space S: The set of all possible states the environment can be in. Can be finite (discrete grid world), countably infinite (Atari game frames), or continuous (robot joint angles ∈ ℝⁿ).
  • A
    Action space A: The set of all possible actions the agent can take. Can be discrete (4 directions) or continuous (velocity vector). Sometimes action space is state-dependent: A(s).
  • P
    Transition function P(s'|s,a): The probability of transitioning to state s' when taking action a in state s. Encodes environment dynamics. In deterministic environments: P(s'|s,a) = 1 for one specific s'. Not known by the agent in model-free RL.
  • R
    Reward function R(s,a,s'): The expected reward when transitioning from s to s' via action a. Often simplified to R(s,a) or R(s). Designed by the RL engineer to encode the task objective.
  • γ
    Discount factor γ ∈ [0,1): Controls the trade-off between immediate and future rewards. Defines the planning horizon.

3.3 Grid World Example — Concrete MDP

  5×5 Grid World MDP:

  ┌────┬────┬────┬────┬────┐
  │ S  │    │    │Wall│    │   S = Start (agent initial position)

    State: (row, col) pair of agent's position. S = {(0,0), (0,1), ..., (4,4)} minus wall cells. |S| = 23.

3.4 Policy, Value Functions

Policy and Value Function Definitions
Policy π(a|s) = P(action=a | state=s) ← stochastic
Policy π(s) = a ← deterministic

State-Value Function V^π(s) = E_π[Gₜ | sₜ=s]
= Expected return starting from state s, following policy π forever

Action-Value Function Q^π(s,a) = E_π[Gₜ | sₜ=s, aₜ=a]
= Expected return starting from state s, taking action a first,
then following policy π forever

Relationship: V^π(s) = Σₐ π(a|s) · Q^π(s,a)

3.5 Optimal Policy and Optimal Value Functions

  • ▸
    Optimal policy π*: The policy that maximizes expected return from every state simultaneously. π* ≥ π for all states s and all policies π — meaning V^{π*}(s) ≥ V^π(s) for all s.
  • ▸
    Optimal state-value V*(s): V*(s) = max_π V^π(s). The maximum expected return achievable from state s under any policy.
  • ▸
    Optimal action-value Q*(s,a): Q*(s,a) = max_π Q^π(s,a). The maximum expected return when taking action a in state s, then acting optimally thereafter.
  • ▸
    Extracting π* from Q*: Given Q*, the optimal policy simply picks the action with the highest Q* value: π*(s) = argmax_a Q*(s,a). This is why solving for Q* (as Q-Learning does) is sufficient — the optimal policy is implicit.

Bellman Equations — The Foundation of Dynamic Programming

4.1 The Bellman Expectation Equations

Richard Bellman (1957) showed that value functions satisfy a fundamental recursive relationship — the current value equals the expected immediate reward plus the discounted value of the next state. This decomposes the problem of finding long-term value into a sequence of one-step-ahead decisions.

Bellman Expectation Equations (for policy π)
Bellman Equation for V^π(s):
V^π(s) = Σₐ π(a|s) Σₛ' P(s'|s,a) [R(s,a,s') + γ·V^π(s')]

= Expected immediate reward + γ × expected value of next state
= "What reward do I get on average right now, plus discounted future?"

Bellman Equation for Q^π(s,a):
Q^π(s,a) = Σₛ' P(s'|s,a) [R(s,a,s') + γ · Σₐ' π(a'|s') · Q^π(s',a')]

Relationship:
V^π(s) = Σₐ π(a|s) · Q^π(s,a)
Q^π(s,a) = Σₛ' P(s'|s,a) [R(s,a,s') + γ·V^π(s')]

4.2 Visual Backup Diagrams

  BELLMAN BACKUP for V(s):                 BELLMAN BACKUP for Q(s,a):

        s                                         s
        │                                         │
     π(a|s)  ← policy                           a (action taken)
    /   |    \                                   │
   a₁   a₂   a₃                            P(s'|s,a)  ← transition
   │    │    │                             /     |     \
  P(s'|s,a)                              s'₁   s'₂   s'₃
  /  \  / \  / \                          │     │     │
 s'  s' s' s' s' s'                   π(a'|s') ← policy
 │               │                   /    \
 R + γV(s')   R + γV(s')         Q(s',a') Q(s',a')

 V(s) = weighted average over all         Q(s,a) = average over transitions,
 action-next state combinations            then average over next actions
Figure 4.1 — Bellman backup diagrams. Circles = states, squares = state-action pairs. Arrows = transitions weighted by probabilities.

4.3 Bellman Optimality Equations

When we restrict to the optimal policy, the Bellman equations become the Bellman Optimality Equations — a set of nonlinear equations whose solution gives the optimal value functions:

Bellman Optimality Equations
Optimal State-Value V*(s):
V*(s) = max_a Σₛ' P(s'|s,a) [R(s,a,s') + γ · V*(s')]

← take the BEST action, then follow optimal policy

Optimal Action-Value Q*(s,a):
Q*(s,a) = Σₛ' P(s'|s,a) [R(s,a,s') + γ · max_{a'} Q*(s',a')]

← transition to s', then take the BEST action from s'

These are nonlinear (due to max operator) and coupled (V* depends on V*).
Cannot be solved analytically → use iterative methods (Value Iteration).
  • ▸
    Key difference from expectation equations: The expectation equation averages over all actions weighted by the policy. The optimality equation takes the maximum over actions — we no longer follow a fixed policy but always do the best thing.
  • ▸
    Unique solution: For finite MDPs with γ < 1, the Bellman optimality equations have a unique solution V*. This is guaranteed by the contraction mapping theorem — the Bellman operator is a contraction.
  • ▸
    Contraction property: The Bellman operator T is a γ-contraction: ‖TV − TV'‖_∞ ≤ γ · ‖V − V'‖_∞. Repeated application shrinks the distance to the fixed point V* by factor γ at each step → convergence guaranteed.
Section 5

Value Iteration

5.1 Core Idea

Value Iteration (Bellman, 1957) is a Dynamic Programming algorithm that finds the optimal value function V* by repeatedly applying the Bellman optimality operator to an initial estimate of V. Since the Bellman operator is a contraction, this iteration converges to V*.

Value Iteration Intuition

Start with V₀(s) = 0 for all states (optimistic — assume everything has zero value). Repeatedly update: "The value of state s is the best I can do — take the best action and receive its immediate reward plus the discounted value of the next state." Each iteration propagates reward information one step further back from terminal states. After enough iterations, value information has propagated from terminals throughout the entire state space.

5.2 Value Iteration Algorithm

Algorithm: Value Iteration
Initialize: V(s) = 0 for all s ∈ S ← arbitrary initialization
Set convergence threshold: ε (e.g. 0.001)

repeat:
  Δ ← 0
  for each s ∈ S:
    v ← V(s) ← save old value
    V(s) ← max_a Σₛ' P(s'|s,a)[R(s,a,s') + γ·V(s')]
    Δ ← max(Δ, |v − V(s)|) ← track max change
until Δ < ε ← converged

Extract policy:
π*(s) = argmax_a Σₛ' P(s'|s,a)[R(s,a,s') + γ·V(s')]

5.4 Convergence Analysis

  • ▸
    Guaranteed convergence: Value Iteration is guaranteed to converge to V* for any finite MDP with γ < 1. The error after k iterations: ‖Vₖ − V*‖_∞ ≤ γᵏ · ‖V₀ − V*‖_∞. Each iteration reduces the error by factor γ.
  • ▸
    Convergence rate: For γ = 0.9, after 100 iterations: error ≤ 0.9^{100} · initial_error ≈ 2.66 × 10⁻⁵ × initial_error. Very fast convergence for small γ, slow for γ close to 1.
  • ▸
    Stopping criterion: Stop when max_s |V_{k+1}(s) − Vₖ(s)| < ε(1−γ)/(2γ). This guarantees the resulting greedy policy is ε-optimal — within ε of the true optimal value.
  • ▸
    Complexity per iteration: O(|S|² · |A|) — must consider every state-action-next-state triple. For large state spaces (e.g., Go: 10^{170} states), exact Value Iteration is computationally infeasible.

5.5 Synchronous vs Asynchronous Value Iteration

  • ▸
    Synchronous (batch) updates: Compute new V(s) for ALL states simultaneously, using old values. Update only after all states processed. Requires storing two copies of V. Guaranteed to converge but may be wasteful.
  • ▸
    Asynchronous (in-place) updates: Update V(s) immediately and use the new value for subsequent state updates in the same sweep. Often converges faster in practice (uses most recent information). Used in practical implementations.
  • ▸
    Prioritized sweeping: Update states in order of their Bellman residual — states with the largest |V_{new} − V_{old}| are updated first. Far fewer total updates needed. Key idea behind prioritized experience replay in DQN.
Section 6

Policy Iteration

6.1 Overview

Policy Iteration is an alternative Dynamic Programming method that alternates between two steps: Policy Evaluation (compute the exact value function for the current policy) and Policy Improvement (make the policy greedy with respect to the computed values). This is guaranteed to converge to the optimal policy in a finite number of iterations.

Algorithm: Policy Iteration
Initialize: π(s) = any fixed action for all s ∈ S

repeat:
  POLICY EVALUATION: find V^π for current π
    repeat:
      Δ ← 0
      for each s ∈ S:
        v ← V(s)
        V(s) ← Σₛ' P(s'|s,π(s))[R(s,π(s),s') + γ·V(s')]
        Δ ← max(Δ, |v − V(s)|)
    until Δ < ε

  POLICY IMPROVEMENT: make π greedy w.r.t. V
  policy_stable ← true
  for each s ∈ S:
    old_action ← π(s)
    π(s) ← argmax_a Σₛ' P(s'|s,a)[R(s,a,s') + γ·V(s')]
    if old_action ≠ π(s): policy_stable ← false

until policy_stable = true ← converged to π*

6.2 Policy Improvement Theorem

  • ▸
    Theorem: If π' is a greedy policy w.r.t. V^π, then V^{π'}(s) ≥ V^π(s) for all s. The new greedy policy is always at least as good as the old policy. Strict improvement unless already at the optimal.
  • ▸
    Proof sketch: V^π(s) ≤ Q^π(s, π'(s)) = Σₛ' P[R + γ·V^π(s')] ≤ ... ≤ V^{π'}(s). The chain of inequalities follows from the greedy improvement and Bellman equation applied recursively.
  • ▸
    Convergence: Since there are finitely many deterministic policies (|A|^|S| for discrete spaces) and each iteration strictly improves (or is already optimal), Policy Iteration must converge in finite iterations. Typically converges in fewer iterations than Value Iteration but each iteration is more expensive (full policy evaluation).

Exam Point — Dynamic Programming for RL

Both Value Iteration and Policy Iteration are Dynamic Programming methods — they require full knowledge of the MDP (transition probabilities P and reward function R). They are model-based, planning algorithms, not learning algorithms. They are only feasible for small, discrete state spaces. For large or unknown state spaces, we need model-free methods — specifically Q-Learning, which does not require knowing P.

Q-Learning — Theory and Motivation

8.1 Moving Beyond Dynamic Programming

Value Iteration requires knowing P(s'|s,a) — the transition probabilities — which are often unavailable in real-world problems. Q-Learning (Watkins, 1989) is a model-free, off-policy temporal-difference method that learns the optimal action-value function Q* directly from experience, without needing a model of the environment.

PropertyValue Iteration (DP)Q-Learning
Requires model?Yes — P(s'|s,a), R(s,a)No — learns from (s,a,r,s') transitions
Tabular storageV-table: |S| entriesQ-table: |S|×|A| entries
Update timingBatch (full sweeps)Online (after every transition)
Policy requirementRequires full MDPOff-policy: learns Q* regardless of behavior policy
FeasibilitySmall discrete spacesLarger spaces; extended by DQN to continuous observation spaces

8.2 Temporal Difference Learning

Q-Learning is a form of Temporal Difference (TD) learning — it updates estimates based on other estimates (bootstrapping), rather than waiting for the final outcome of an episode (Monte Carlo methods).

TD Learning Insight — Learn While Playing

Monte Carlo methods wait until the end of an episode to update, using the actual return Gₜ as the target. TD methods update at every step using a bootstrapped target: r + γ·V(s'). They learn online from incomplete episodes. This is much more sample-efficient and allows learning in continuing (non-episodic) tasks.

TD Error (δ): δ = [r + γ·V(s')] − V(s). The difference between the estimated value and the better estimate from one step later. This is the "surprise" — how wrong was our current estimate?

8.3 Q-Learning — The Update Rule

Q-Learning Update Rule (Watkins & Dayan, 1992)
Q(s,a) ← Q(s,a) + α · δ

where the TD error δ is:
δ = [ r + γ · max_{a'} Q(s',a') ] − Q(s,a)

Full update:
Q(sₜ,aₜ) ← Q(sₜ,aₜ) + α · [ rₜ₊₁ + γ · max_{a'} Q(sₜ₊₁,a') − Q(sₜ,aₜ) ]

← Target = rₜ₊₁ + γ·max_{a'} Q(s',a') (Bellman optimality backup)
← Current est. = Q(sₜ,aₜ)
← TD error δ = Target − Current estimate
← α (0<α≤1) = learning rate (step size)
  • ▸
    The target r + γ·max_a' Q(s',a'): This is the Bellman optimality backup evaluated using the current (imperfect) Q estimates. As Q improves, the targets become more accurate, pulling Q toward Q*.
  • ▸
    max operator — off-policy: The update uses max_a' Q(s',a') regardless of what action was actually taken next. This is what makes Q-Learning off-policy — it learns about the greedy (optimal) policy even when following an exploratory behavior policy (like ε-greedy).
  • ▸
    Learning rate α: Controls how much new information overwrites old estimates. α=1: fully replace old estimate with new target. α→0: barely update. Typically α=0.1 or decayed over time. Must satisfy Σα = ∞, Σα² < ∞ for convergence guarantees.
  • ▸
    Bootstrapping: The target uses the current Q estimate for s' — this is bootstrapping. The target is not ground truth but an estimate from the current Q function. Q-Learning is therefore biased but has low variance (cf. Monte Carlo: unbiased, high variance).

8.4 SARSA vs Q-Learning — On-Policy vs Off-Policy

SARSA (On-Policy) vs Q-Learning (Off-Policy)
SARSA update:
Q(s,a) ← Q(s,a) + α[r + γ·Q(s',a') − Q(s,a)]
← uses the ACTUAL next action a' that was taken (from behavior policy)

Q-Learning update:
Q(s,a) ← Q(s,a) + α[r + γ·max_{a'} Q(s',a') − Q(s,a)]
← uses the BEST possible next action (regardless of what was taken)

SARSA learns Q^π for the current behavior policy π (on-policy).
Q-Learning learns Q* directly, regardless of behavior policy (off-policy).

Consequence: Q-Learning converges to optimal even with exploratory behavior. SARSA converges to policy-dependent values — if you explore, SARSA accounts for the cost of exploration; Q-Learning assumes you will always act optimally.
Section 9

Q-Learning Algorithm — Complete

9.1 Tabular Q-Learning

Algorithm: Tabular Q-Learning (Watkins, 1989)
Initialize: Q(s,a) = 0 for all s ∈ S, a ∈ A (or small random values)
Hyperparameters: α (learning rate), γ (discount), ε (exploration)

for each episode:
  Initialize state s ← s₀ (start state)

  for each step t of episode:
    Choose action a using ε-greedy policy:
      with prob ε: a ← random action ← EXPLORE
      with prob (1-ε): a ← argmax_{a'} Q(s,a') ← EXPLOIT

    Take action a, observe reward r and next state s'

    Update Q-table:
    Q(s,a) ← Q(s,a) + α · [r + γ · max_{a'} Q(s',a') − Q(s,a)]

    s ← s' ← transition to next state

    if s is terminal: break

until Q converges or budget exhausted

9.2 Convergence of Q-Learning

  • ▸
    Convergence theorem (Watkins & Dayan, 1992): Tabular Q-Learning converges to Q* with probability 1 given: (1) All state-action pairs are visited infinitely often. (2) Learning rate α satisfies Σₜ αₜ = ∞ and Σₜ αₜ² < ∞ (e.g. αₜ = 1/t). (3) Rewards are bounded. (4) Finite MDP with γ < 1.
  • ▸
    Why visit all pairs? If some (s,a) pairs are never sampled, their Q values remain at initialization and may be incorrect. The agent must explore sufficiently to guarantee all pairs are tried — hence the exploration requirement.
  • ▸
    Practical α: In practice, a fixed small α (e.g. 0.1) works well even without the theoretical decay. The learning rate condition is a theoretical requirement for exact convergence proofs, not always necessary in practice.

9.3 Q-Learning on Grid World — Trace

  Q-TABLE EVOLUTION for Grid World (simplified, γ=0.9, α=0.5):

  Q(s,a) = Q-value for state s, action a

  Initial Q-table:  All zeros

  EPISODE 1 (random policy, reaches GOAL at step 5):
  Step 1: s=(0,0), a=Right, r=-0.04, s'=(0,1)
    Q((0,0),Right) ← 0 + 0.5·[-0.04 + 0.9·max Q((0,1),·) − 0]
                  ← 0 + 0.5·[-0.04 + 0] = -0.02

  Step 4: s=(0,3), a=Right, r=+1 (GOAL), s'=terminal
    Q((0,3),Right) ← 0 + 0.5·[+1 + 0.9·0 − 0] = 0.5

  EPISODE 2 (explores, finds goal again):
    Same (0,3),Right: Q ← 0.5 + 0.5·[1 + 0 − 0.5] = 0.75

  AFTER MANY EPISODES (Q has converged):
  Q-table will reflect optimal action-values:
  High Q values for actions that efficiently reach the goal,
  low or negative Q values for actions toward the trap.

  OPTIMAL POLICY extracted: π*(s) = argmax_a Q*(s,a)
  → Same as Value Iteration result, but learned purely from experience!
Figure 9.1 — Q-Learning trace showing how Q-values update from experience. Values propagate backwards from the goal through repeated episodes.

9.4 Q-Learning Limitations

  • ▸
    Curse of dimensionality: Tabular Q-Learning stores one Q value per (state, action) pair. For Atari: 84×84×4 frames → ~10^{18,000} states. For a robot: continuous state space with uncountably infinite states. A table cannot store these.
  • ▸
    No generalization: Q-table treats every state independently. Learning Q((0,0), Right) provides zero information about Q((0,1), Right). Yet in many problems, nearby states have similar values. A function approximator (neural network) can generalize across similar states.
  • ▸
    Slow convergence for large spaces: Even if we could store a large Q-table, visiting every state-action pair enough times for convergence would take astronomically many episodes for large state spaces.
  • ▸
    Solution: Replace the Q-table with a neural network Q_θ(s,a) — this is Deep Q-Network (DQN).
Section 10

Exploration vs Exploitation

10.1 The Fundamental Dilemma

The exploration-exploitation trade-off is the central challenge in RL. The agent must balance:

  • ▸
    Exploitation: Choose the action with the highest known Q-value. Maximize immediate expected reward based on current knowledge. Problem: misses potentially better actions that haven't been tried enough — greedy exploitation gets stuck in local optima.
  • ▸
    Exploration: Try new, uncertain actions to gather more information about the environment. Required for finding the optimal policy. Problem: wastes time on bad actions when the agent could be exploiting known good actions.

10.2 Exploration Strategies

A. ε-Greedy (Most Common)

ε-Greedy Policy
π_ε(a|s) = 1 − ε + ε/|A| if a = argmax_{a'} Q(s,a') ← greedy action
= ε / |A| otherwise ← random action

Simplified: with prob ε take random action, with prob (1−ε) take greedy action

ε-decay schedule: εₜ = max(ε_min, ε₀ · decay^t)
e.g. ε₀=1.0, decay=0.995, ε_min=0.01 → Start fully random (explore), gradually become more greedy (exploit)
  • ▸
    ε-decay: Start with high ε (fully exploratory, know nothing) and decrease over time (mostly exploit learned knowledge). The decaying schedule matches the agent's increasing knowledge — explore when uncertain, exploit when confident.
  • ▸
    Problem with ε-greedy: Treats all non-greedy actions equally. A slightly-suboptimal action is explored as frequently as a clearly terrible action. Does not focus exploration on uncertain or high-potential actions.

B. Upper Confidence Bound (UCB)

UCB Action Selection
a* = argmax_a [ Q(s,a) + c · √(ln t / Nₜ(a)) ]

Nₜ(a) = number of times action a has been taken so far
c = exploration constant (higher = more exploration)
t = total timesteps so far

Intuition: Take the action with the highest OPTIMISTIC estimate. √(ln t / Nₜ(a)) is large when a has been tried rarely (high uncertainty) → UCB naturally focuses exploration on rarely-tried, uncertain actions.

C. Boltzmann (Softmax) Exploration

  • ▸
    Formula: π(a|s) = exp(Q(s,a)/τ) / Σ_{a'} exp(Q(s,a')/τ). Temperature τ controls randomness: high τ → uniform (explore), low τ → greedy (exploit).
  • ▸
    Advantage: Actions are selected with probability proportional to their Q-value — better actions are tried more often, worse actions less often. More principled than ε-greedy which samples uniformly among non-greedy actions.

D. Intrinsic Motivation (Curiosity)

  • ▸
    Idea: Add an intrinsic reward bonus for visiting novel, surprising states. Curiosity-Driven Exploration: the reward is prediction error of a state dynamics model — novel states the model can't predict well get high intrinsic reward.
  • ▸
    Applications: Sparse reward environments where external reward is very rare. Encourages the agent to systematically explore the state space rather than staying near familiar states.
Part 5 — Deep Q-Network (DQN)
Section 11

Deep Q-Network (DQN)

11.1 From Tabular to Deep Q-Learning

DQN (Mnih et al., DeepMind — 2013/2015) replaces the Q-table with a deep neural network Q_θ(s,a) that takes a state s as input and outputs Q-values for all actions simultaneously. This extends Q-Learning to high-dimensional, continuous state spaces (e.g., raw pixel images from Atari games).

DQN's Historic Achievement

DQN was the first algorithm to learn to play Atari games from raw pixels (no hand-crafted features), achieving superhuman performance on 49 games with a single algorithm and architecture. The same network, hyperparameters, and training procedure was used for all games — only the scores changed. This demonstrated that deep RL could learn complex control policies directly from high-dimensional sensory input.

11.2 DQN Architecture

  DQN for Atari Games:

  INPUT: 4 consecutive game frames (84×84 pixels, grayscale, stacked)
  Shape: 84 × 84 × 4  (4-frame stack captures motion)
       │

  │  CONVOLUTIONAL LAYERS (feature extraction)                   │
  │  Conv1: 32 filters, 8×8, stride 4  → 20×20×32  + ReLU       │
  │  Conv2: 64 filters, 4×4, stride 2  → 9×9×64    + ReLU       │

       │

  Output: Q_θ(s, a₁), Q_θ(s, a₂), ..., Q_θ(s, a_{|A|})
  All action values in ONE forward pass!
  Action selection: a* = argmax_a Q_θ(s,a)
Figure 11.1 — DQN architecture for Atari. Convolutional layers extract visual features; FC layers compute Q-values for all actions.

11.3 The DQN Loss Function

DQN Loss — Mean Squared Bellman Error
L(θ) = E_{(s,a,r,s') ~ D} [ (y − Q_θ(s,a))² ]

where the TD target y:
y = r + γ · max_{a'} Q_{θ⁻}(s', a') ← θ⁻ = target network params

θ ← θ − α · ∇_θ L(θ)

Note: gradient is NOT propagated through the target y. The target y is treated as a fixed constant during each update.

11.4 Two Key Innovations — Why DQN Works

A. Experience Replay

  • ▸
    The problem without replay: Consecutive transitions (s₁,a₁,r₁,s₁') and (s₂,a₂,r₂,s₂') are highly correlated (s₂ ≈ s₁'). If we train on consecutive transitions, the network overfits to recent experience and "forgets" earlier transitions — catastrophic interference.
  • ▸
    Experience Replay buffer D: Store all transitions (s,a,r,s') in a replay buffer of capacity N (e.g. 1 million). At each training step, sample a random mini-batch of 32–64 transitions from D. Train on this random sample.
  • ▸
    Benefits: (1) Breaks temporal correlations between consecutive samples → each mini-batch is approximately i.i.d. (2) Every transition can be learned from many times (high data efficiency). (3) Decouples the frequency of environment interaction from the frequency of learning steps.
  • ▸
    Off-policy requirement: Experience Replay requires the algorithm to be off-policy (Q-Learning qualifies). The stored transitions were generated by older policy versions — an on-policy method cannot use stale data.

B. Target Network

  • ▸
    The moving target problem: In Q-Learning, the target y = r + γ·max_a' Q(s',a'). With a neural network, every weight update changes Q_θ for all states — including s'. This means the target y changes after every update, creating a moving target that can lead to oscillations or divergence.
  • ▸
    Target network Q_{θ⁻}: Maintain a separate copy of the Q-network with frozen parameters θ⁻. Use Q_{θ⁻}(s',a') to compute targets. Update θ⁻ ← θ only every C steps (e.g. C=10,000). This stabilizes targets for thousands of gradient steps between updates.
  • ▸
    Effect: The target y = r + γ·max_a' Q_{θ⁻}(s',a') is now stable for C steps. The online network Q_θ chases this stable target. When C steps pass, θ⁻ is refreshed to the current θ and the process repeats.
  • ▸
    Soft update alternative: θ⁻ ← τ·θ + (1−τ)·θ⁻ at every step (Polyak averaging), τ=0.001. Slower, smoother update of target network. Used in DDPG, TD3, SAC.
  DQN TRAINING LOOP:

  Environment    Replay Buffer D         Online Net Q_θ      Target Net Q_{θ⁻}
       │                │                     │                    │
  ─────┼────────────────┼─────────────────────┼────────────────────┼────────
       │                │                     │                    │
  observe sₜ            │                     │                    │
       │                │                     │                    │

       │                │                     │                    │
  execute a, get r,s'   │                     │                    │

  {(sᵢ,aᵢ,rᵢ,s'ᵢ)}     │                     │                    │

  every C steps: copy θ⁻ ← θ ────────────────────────────────────▶ θ⁻

Section 12

DQN Improvements

12.1 Double DQN (DDQN) — Fixing Overestimation

  • ▸
    Problem — Maximization bias: DQN uses max_a' Q_{θ⁻}(s',a') for the target. The max operation systematically overestimates Q-values. If any Q-values have random noise (they always do), the max picks the one with the highest noise — consistently overestimating the true maximum.
  • ▸
    DDQN fix: Decouple action selection from action evaluation. Use the ONLINE network to SELECT the best action: a* = argmax_a Q_θ(s',a). Use the TARGET network to EVALUATE it: Q_{θ⁻}(s', a*). The target becomes: y = r + γ · Q_{θ⁻}(s', argmax_a Q_θ(s',a)).
  • ▸
    Result: Significantly reduces overestimation bias. More accurate Q-value estimates. Better performance on most Atari games.

12.2 Dueling DQN — Decomposing Q-Values

  • ▸
    Insight: Q(s,a) = V(s) + A(s,a). State value V(s) measures how good it is to BE in state s (regardless of action). Advantage A(s,a) measures how much BETTER action a is compared to the average action in s. Decomposing helps learning — V can be updated from all actions, not just the one taken.
  • ▸
    Architecture: After the convolutional layers, split into two streams: a value stream V(s) (outputs scalar) and an advantage stream A(s,a) (outputs |A|-dim vector). Combined: Q(s,a) = V(s) + (A(s,a) − mean_a' A(s,a')). The mean subtraction ensures identifiability (prevents V and A from being ambiguous).
  • ▸
    Benefit: For states where all actions are equally good (e.g., a racing game going straight), V is updated meaningfully without needing to evaluate each specific action. Faster learning in such states.

12.3 Prioritized Experience Replay (PER)

  • ▸
    Problem with uniform replay: Not all transitions are equally informative. Transitions with high TD error δ contain more surprise — the Q estimate was more wrong for these. Sampling them more often would accelerate learning.
  • ▸
    PER mechanism: Sample transition i with probability P(i) ∝ |δᵢ|^α. Higher priority (larger TD error) → sampled more often. α=0 → uniform. α=1 → fully proportional. Typically α=0.6.
  • ▸
    Importance sampling correction: PER introduces bias (over-samples high-error transitions). Correct with importance sampling weights wᵢ = (1/N · 1/P(i))^β. β annealed from 0.4 to 1 over training. Prevents bias from dominating in later training.

12.4 Rainbow DQN

Rainbow (Hessel et al., DeepMind — 2017) combines six DQN improvements into one agent and demonstrates that all six components contribute to performance:

1. Double DQN

Reduces overestimation bias by decoupling action selection and evaluation.

2. Dueling Networks

Separate value and advantage streams for more efficient value estimation.

3. Prioritized Replay

Sample high-TD-error transitions more frequently.

4. Multi-step Returns

Use n-step TD targets instead of 1-step. Better credit assignment. n=3 typical.

5. Distributional RL (C51)

Learn full distribution of returns, not just expected value. Predicts 51 atoms of the return distribution.

6. Noisy Networks

Learnable noise in network weights for exploration. Replaces ε-greedy with learned, state-dependent exploration.

Exam Point — Off-Policy vs On-Policy

On-policy: Learns about the policy being used to collect data. Must use fresh data — cannot reuse old transitions. SARSA, REINFORCE, PPO. Safe in risky environments because it accounts for the cost of exploration.

Off-policy: Learns about a different (usually optimal/greedy) policy than the one used to collect data. Can reuse old transitions (experience replay). Q-Learning, DQN, SAC. More data-efficient but can be less stable.

Key Q-Learning insight: Q-Learning is off-policy because it uses max_a' Q(s',a') (greedy policy) in the target regardless of what action a' was actually taken. This lets it learn the optimal Q* even while exploring randomly.

Module 4 — Complete Summary

  • 1.
    Reinforcement Learning: Agent learns by interacting with environment — observes state, takes action, receives scalar reward, transitions to new state. Goal: find policy π that maximizes discounted return Gₜ = Σ γᵏrₜ₊ₖ. No labeled data — learns from experience via trial and error.
  • 2.
    MDP = (S, A, P, R, γ): Markov Property — future depends only on current state. V^π(s) = expected return from s following π. Q^π(s,a) = expected return taking action a then following π. Optimal: V*(s) = max_π V^π(s). π*(s) = argmax_a Q*(s,a).
  • 3.
    Bellman Equations: V^π(s) = Σ_a π(a|s) Σ_{s'} P(s'|s,a)[R + γV^π(s')]. Optimality: V*(s) = max_a Σ_{s'} P(s'|s,a)[R + γV*(s')]. The max operator makes these nonlinear — solved iteratively.
  • 4.
    Value Iteration: Repeatedly apply Bellman optimality: V_{k+1}(s) ← max_a Σ P[R+γVₖ(s')]. Converges to V* because Bellman operator is γ-contraction. Requires full MDP knowledge. Complexity O(|S|²|A|) per iteration.
  • 5.
    Policy Iteration: Alternates Policy Evaluation (compute V^π fully) and Policy Improvement (π←greedy(V^π)). Converges in finite iterations (Policy Improvement Theorem). Each iteration more expensive but fewer iterations than VI.
  • 6.
    Q-Learning: Model-free, off-policy TD method. Update: Q(s,a) ← Q(s,a) + α[r + γ·max_a' Q(s',a') − Q(s,a)]. Uses max (greedy) regardless of behavior policy → learns Q* directly. Requires all (s,a) pairs visited infinitely often for convergence.
  • 7.
    Exploration: ε-greedy (explore with prob ε, decay over time), UCB (optimistic bonus ∝ 1/√visits), Boltzmann (probability ∝ exp(Q/τ)), Intrinsic motivation (reward novelty). Exploration-exploitation trade-off is fundamental to RL.
  • 8.
    DQN: Replaces Q-table with CNN Q_θ(s,a). Two key innovations: (1) Experience Replay — random mini-batch from replay buffer D breaks correlations. (2) Target Network Q_{θ⁻} — frozen copy updated every C steps stabilizes targets. Achieved superhuman Atari performance from raw pixels.
  • 9.
    DQN Improvements: Double DQN (decouple selection/evaluation, reduce overestimation). Dueling DQN (Q=V+A, two streams). Prioritized Replay (sample by |δ|). Rainbow = all six improvements combined → state-of-art on Atari.
