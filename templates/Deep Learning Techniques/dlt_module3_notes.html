Section 1

Motivation for Attention

1.1 The Bottleneck Problem in Seq2Seq

Before attention, sequence-to-sequence (Seq2Seq) models used an encoder-decoder RNN. The encoder compressed the entire input sequence into a single fixed-size context vector c — the encoder's final hidden state. The decoder used only this one vector to generate the entire output sequence.

  TRADITIONAL SEQ2SEQ (without attention):

  Input: "The cat sat on the mat"
         ↓   ↓   ↓   ↓   ↓   ↓
        [h₁]→[h₂]→[h₃]→[h₄]→[h₅]→[h₆] → c (single fixed vector)
                                                 ↓
                                         [Decoder uses c only]
                                                 ↓
                                    Output: "Le chat s'est assis..."

  PROBLEM:
  • c must encode ALL information from the input sequence
  • For long sequences, early information is "forgotten"
  • Decoder has no way to focus on specific input words
  • Performance degrades sharply with sequence length
Figure 1.1 — Seq2Seq bottleneck. A single context vector cannot adequately summarize long sequences.

1.2 The Attention Solution

Bahdanau et al. (2015) proposed a revolutionary idea: instead of using only the final hidden state, let the decoder attend to all encoder hidden states at each decoding step, assigning a learned attention weight to each. The decoder can "look back" at any relevant part of the input.

Core Attention Intuition

When translating "Le chat" (the cat), the decoder should pay most attention to the encoder's representation of "The cat" — not "on the mat." Attention learns which encoder states to focus on for each decoder step. This focus is dynamic — it changes at every output token based on what is being generated.

Attention replaces the bottleneck with a dynamic, weighted combination of ALL encoder states. The decoder gets a rich, position-aware context vector tailored to each output step.

1.3 The General Attention Principle

  • ▸
    Selective focus: Not all input positions are equally relevant for every computation. Attention allows the model to focus on what matters, ignoring the rest. This mimics human reading — we don't re-read everything to understand one word; we focus on the relevant context.
  • ▸
    Dynamic weighting: Attention weights are computed dynamically at each step based on the current query (what we're looking for) and the available context (what's in memory). Different steps attend to different positions.
  • ▸
    Parallelization: Unlike RNNs that must process sequences sequentially, attention operations can be computed for all positions simultaneously — a massive advantage for GPU/TPU training.
  • ▸
    Interpretability: Attention weights provide a window into what the model is "looking at" — a form of built-in interpretability not available in RNNs or CNNs.
Section 2

Query, Key, Value Framework

2.1 The Database Analogy

The Query-Key-Value (QKV) framework is best understood through a database analogy. Imagine a database of information where you search using a query:

  DATABASE ANALOGY                        ATTENTION ANALOGY

  Query: "find me articles about cats"    Q: what information am I looking for?
                                          (current decoder state, or current token)
  Keys: titles of all documents           K: what does each memory entry "contain"?
  [ "Dog care", "Cat genetics",  ]        (summaries / labels of each encoder state)
    "Fish tanks", "Cat behavior" ]

  Exact lookup:                           Soft (fuzzy) lookup:
  → returns "Cat genetics" only           → returns a WEIGHTED COMBINATION
  (hard match, key = query)               (all entries, weights based on similarity)

  Values: full content of each doc        V: actual information in each memory entry
  → returns full text of "Cat genetics"   → weighted sum of all encoder states

  KEY INSIGHT:
  Hard database: returns ONE document (binary)
  Attention: returns weighted COMBINATION of ALL documents (soft / differentiable)
Figure 2.1 — QKV database analogy. Attention performs a soft, differentiable lookup rather than a hard match.

2.2 Formal Definitions

  • ▸
    Query (Q): Represents "what am I looking for?" In encoder-decoder attention: the current decoder hidden state (what the decoder needs to produce the next output word). In self-attention: the current token representation (what it wants to learn from context). Dimension: d_q.
  • ▸
    Key (K): Represents "what do I contain / what information does this entry offer?" Each position in the input sequence has a key — a representation of what information that position can provide. Similarity between Q and K determines attention weight. Dimension: d_k (must equal d_q for dot product).
  • ▸
    Value (V): Represents "the actual information at this position." After computing attention weights from Q-K similarity, the output is a weighted sum of values. Values can have a different dimension from keys: d_v.
  • ▸
    Why separate K and V? Keys determine HOW MUCH to attend (attention weights). Values determine WHAT to extract (the actual information). Decoupling them allows the model to use different representations for "searching" (K) and "retrieving" (V). More expressive than using the same vector for both.

2.3 From Input to Q, K, V — Linear Projections

In transformers, Q, K, and V are not the raw input vectors. They are produced by learnable linear projections of the input:

Linear Projections to Q, K, V
Input: X (sequence of n token embeddings, each d_model dimensional)
X ∈ ℝ^{n × d_model}

Q = X · W^Q where W^Q ∈ ℝ^{d_model × d_k} ← query projection
K = X · W^K where W^K ∈ ℝ^{d_model × d_k} ← key projection
V = X · W^V where W^V ∈ ℝ^{d_model × d_v} ← value projection

W^Q, W^K, W^V are LEARNABLE parameters — the model learns the best projections. Different projections = different "views" of the same input for searching (K), querying (Q), and extracting (V).
  • ▸
    Same input, three purposes: In self-attention, X is used to generate all three: Q, K, and V. The same token projects itself into what it's looking for (Q), what it offers (K), and what it provides (V).
  • ▸
    Typically d_k = d_v = d_model / h where h is the number of attention heads. Standard Transformer: d_model=512, h=8 → d_k=d_v=64 per head.
  • ▸
    Learned projections = learned feature spaces: The model learns which aspects of token representations are most useful for computing compatibility (K, Q) vs which aspects carry the most useful information to propagate (V).
Section 3

Attention Pooling

3.1 What is Attention Pooling?

Attention pooling is the operation of computing a weighted average (or sum) of values, where the weights are derived from compatibility scores between a query and keys. It is the core computational primitive behind all modern attention mechanisms.

3.2 Nadaraya-Watson Kernel Regression — The Origin

Attention pooling has deep roots in classical statistics. The Nadaraya-Watson estimator (1964) predicts an output as a weighted average of training outputs, where weights depend on input similarity:

Nadaraya-Watson Kernel Regression (1964)
f(x) = Σᵢ α(x, xᵢ) · yᵢ

α(x, xᵢ) = K(x − xᵢ) / Σⱼ K(x − xⱼ) ← attention weight (normalized)
K(u) = kernel function (e.g. Gaussian: exp(−u²/2))

x = query (what we want to predict for)
xᵢ = keys (training input positions)
yᵢ = values (training output values)

This is EXACTLY attention: • Compute similarity between query x and each key xᵢ via kernel K • Normalize to get weights α (using softmax-like normalization) • Output = weighted sum of values yᵢ

3.3 Additive Attention (Bahdanau, 2015)

The first neural attention mechanism, introduced for machine translation:

Additive Attention (Bahdanau Attention)
score(q, k) = v^T · tanh(W_q · q + W_k · k) ← compatibility score

α(q, K) = softmax(score(q, k₁), score(q, k₂), ..., score(q, kₙ))

output = Σᵢ αᵢ · vᵢ ← weighted sum of values

Parameters: W_q (d_q × d), W_k (d_k × d), v (d × 1) Called "additive" because q and k are combined by ADDITION (after projection)

3.4 General Attention Pooling Formula

General Attention Pooling
Attention(Q, K, V) = softmax( score(Q, K) ) · V

Step 1: Compute scores: score(Q, K) → shape (n_q × n_k)
How compatible is each query with each key?

Step 2: Normalize: α = softmax(scores) → shape (n_q × n_k)
Convert scores to probability distributions (sum to 1 across keys)

Step 3: Aggregate: output = α · V → shape (n_q × d_v)
Each query gets a weighted sum of all values

3.5 Different Score Functions

Score FunctionFormulaParametersProperties
Additive (Bahdanau) vᵀ tanh(Wq·q + Wk·k) Wq, Wk, v Flexible, handles different dims for Q and K. More parameters. Slower.
Dot Product qᵀ · k None Fast, no parameters. Requires d_q = d_k. Scales with dimension → saturation.
Scaled Dot Product qᵀ · k / √d_k None Dot product with scaling. Prevents softmax saturation. Standard in Transformers.
Cosine qᵀ·k / (‖q‖·‖k‖) None Scale-invariant. Used in some retrieval tasks. Bounded to [−1, 1].
Bilinear (General) qᵀ · W · k W (d_q × d_k) Learned compatibility matrix. Can handle different Q/K dims. Used in Luong attention.
Exam Point — Why Scale by √d_k?

The dot product q·k = Σᵢ qᵢkᵢ. If q and k have components drawn from N(0,1), the dot product has variance = d_k (grows with dimension). For large d_k (e.g. 64), dot products become very large → softmax receives large inputs → its gradient becomes very small (saturation in the flat region) → vanishing gradients. Scaling by √d_k normalizes the variance back to ~1, keeping softmax in its sensitive, high-gradient regime.

Section 4

Scaled Dot-Product Attention — Full Derivation

4.1 The Operation

Scaled Dot-Product Attention — Complete Formula
Attention(Q, K, V) = softmax( Q · Kᵀ / √d_k ) · V

Dimensions:
Q ∈ ℝ^{n_q × d_k} ← n_q queries, each d_k dimensional
K ∈ ℝ^{n_k × d_k} ← n_k keys, each d_k dimensional
V ∈ ℝ^{n_k × d_v} ← n_k values, each d_v dimensional

Q·Kᵀ: (n_q × d_k) · (d_k × n_k) = (n_q × n_k) ← attention score matrix
/√d_k: element-wise scaling ← prevent saturation
softmax: row-wise softmax → each row sums to 1 ← attention weights
·V: (n_q × n_k) · (n_k × d_v) = (n_q × d_v) ← output

Output has n_q rows (one output per query), each d_v dimensional.

4.2 Step-by-Step Walkthrough with Concrete Example

  Sequence: ["I", "love", "deep", "learning"]   (n=4 tokens)
  d_k = d_v = 3 (simplified for illustration)

  After W^Q, W^K, W^V projections:
  Q = [[0.2, 0.5, 0.1],    K = [[0.3, 0.4, 0.2],    V = [[v₁₁, v₁₂, v₁₃],
       [0.8, 0.1, 0.3],         [0.1, 0.9, 0.1],         [v₂₁, v₂₂, v₂₃],
       [0.4, 0.7, 0.2],         [0.6, 0.3, 0.5],         [v₃₁, v₃₂, v₃₃],
       [0.1, 0.3, 0.9]]         [0.2, 0.1, 0.8]]          [v₄₁, v₄₂, v₄₃]]

  Step 1: Q·Kᵀ = (4×3)·(3×4) = 4×4 raw score matrix
  ┌─────────────────────────────────────────────────┐
  │         "I"    "love"  "deep"  "learn."         │
  │ "I"    [s₁₁    s₁₂     s₁₃     s₁₄  ]          │
  │ "love" [s₂₁    s₂₂     s₂₃     s₂₄  ]          │
  │ "deep" [s₃₁    s₃₂     s₃₃     s₃₄  ]          │
  │ "learn"[s₄₁    s₄₂     s₄₃     s₄₄  ]          │
  └─────────────────────────────────────────────────┘
  sᵢⱼ = how much token i should attend to token j

  Step 2: Divide by √3 ≈ 1.73 (scale)

  Step 3: Row-wise softmax → attention weight matrix α
  Each row = attention distribution over all positions for that query token

  Step 4: α · V = (4×4)·(4×3) = (4×3) output
  Each output row = weighted combination of value vectors
Figure 4.1 — Scaled dot-product attention step by step with 4-token sequence.

4.3 Optional Masking

  • ▸
    Padding mask: Input sequences are padded to equal length in a batch. Padding tokens should not contribute to attention. Set their scores to −∞ before softmax → exp(−∞) = 0 → zero attention weight on padding positions.
  • ▸
    Causal (look-ahead) mask: Used in the decoder's self-attention. Token at position i should not attend to positions j > i (future tokens that haven't been generated yet). Set upper triangle of the score matrix to −∞ before softmax. This makes the Transformer autoregressive (generates left-to-right).
Masked Scaled Dot-Product Attention
Attention(Q, K, V, mask) = softmax( (Q·Kᵀ/√d_k) + mask ) · V

Padding mask: mask[i,j] = 0 if j is valid, −∞ if j is padding

Causal mask: mask[i,j] = 0 if j ≤ i (can attend), −∞ if j > i (future)
┌─────────────────────┐
│ 0 −∞ −∞ −∞ │ ← token 1 attends only to itself
│ 0 0 −∞ −∞ │ ← token 2 attends to 1,2
│ 0 0 0 −∞ │ ← token 3 attends to 1,2,3
│ 0 0 0 0 │ ← token 4 attends to all
└─────────────────────┘
Section 5

Multi-Head Attention

5.1 Motivation — Why Multiple Heads?

A single attention function computes ONE weighted combination based on ONE query-key compatibility measure. But relationships in language (and vision) are complex and multi-faceted. A word can relate to others syntactically (subject-verb agreement), semantically (synonymy), and pragmatically (discourse structure) — all simultaneously.

Multi-Head Attention Insight

Multi-head attention runs h attention operations in parallel, each with its own learned Q, K, V projections. Each "head" attends to different aspects of the input simultaneously. Head 1 might learn syntactic dependencies. Head 2 might learn semantic similarity. Head 3 might learn coreference. The outputs of all heads are concatenated and linearly projected, giving a rich representation that integrates all these relationship types at once.

5.2 Multi-Head Attention — Complete Formula

Multi-Head Attention
MultiHead(Q, K, V) = Concat(head₁, head₂, ..., headₕ) · W^O

where each head is:
headᵢ = Attention(Q·W^Q_i, K·W^K_i, V·W^V_i)
= softmax( (Q·W^Q_i)(K·W^K_i)ᵀ / √d_k ) · (V·W^V_i)

Projection dimensions (standard Transformer, h=8, d_model=512):
W^Q_i ∈ ℝ^{d_model × d_k} d_k = d_model / h = 512/8 = 64
W^K_i ∈ ℝ^{d_model × d_k} d_v = d_model / h = 64
W^V_i ∈ ℝ^{d_model × d_v}
W^O ∈ ℝ^{h·d_v × d_model} = ℝ^{512 × 512} ← output projection

Total parameters per MHA layer:
4 × d_model² = 4 × 512² = 1,048,576 (for h=8, d_model=512)

5.3 Multi-Head Attention Architecture Diagram

  Input Q, K, V (each n × d_model)
       │
  ┌────┴────────────────────────────────────────────────────────────────┐
  │                 SPLIT INTO h PARALLEL HEADS                        │
  ├────────────────┬────────────────┬────────────────┬────────────────┤
  │    Head 1      │    Head 2      │    Head 3      │    Head h      │
  │ W^Q_1, W^K_1  │ W^Q_2, W^K_2  │ W^Q_3, W^K_3  │ W^Q_h, W^K_h  │
  │     W^V_1     │     W^V_2     │     W^V_3     │     W^V_h     │
  │       ↓        │       ↓        │       ↓        │       ↓        │
  │ ScaledDotAttn  │ ScaledDotAttn  │ ScaledDotAttn  │ ScaledDotAttn  │
  │   n × d_k      │   n × d_k      │   n × d_k      │   n × d_k      │
  └────────────────┴────────────────┴────────────────┴────────────────┘
                                  │
              ┌─────────── Concat (n × h·d_k = n × d_model)
              │
          [W^O projection]    (d_model × d_model)
              │
         Output: n × d_model
Figure 5.1 — Multi-head attention. h parallel attention heads, each with own projections, outputs concatenated and projected.

5.4 What Different Heads Learn

  • ▸
    Empirical findings (Voita et al., 2019): Analyzing trained BERT heads found interpretable patterns. Some heads focus on syntactic relations (subject→verb). Others on co-reference (pronoun→referent). Others on sequential ordering (attending to the next/previous token). Still others attend broadly to the whole sequence.
  • ▸
    Head specialization: Different layers' heads learn different things. Early layer heads tend to focus on local syntactic structure. Later layer heads capture long-range semantic dependencies.
  • ▸
    Head pruning: Most heads can be pruned (removed) with minimal performance loss — suggesting that a few heads carry most of the useful information. Typically 20–30% of heads can be pruned without significant accuracy drop.

5.5 Computational Complexity

  • ▸
    Time complexity: O(n² · d). The Q·Kᵀ multiplication is (n×d)·(d×n) = n² · d operations. For a sequence of length n, attention scales quadratically in n — the fundamental bottleneck for long sequences.
  • ▸
    Space complexity: O(n²) to store the attention matrix. For n=1000 tokens, the matrix is 1000×1000 = 1M entries. For n=100,000 (long documents), 10B entries — infeasible with standard attention.
  • ▸
    Efficient attention variants: Sparse attention (attend to only local neighbors + global tokens), Linformer (low-rank approximation), Performer (random feature approximation), FlashAttention (IO-aware exact computation). All aim to reduce the n² bottleneck.

Positional Encoding

6.1 Why Positional Encoding is Needed

Attention is permutation invariant — swapping the order of tokens in the input doesn't change the attention output (just reorders it). But order is crucial: "The dog bit the man" ≠ "The man bit the dog." Without explicitly encoding position, the Transformer cannot distinguish these.

6.2 Sinusoidal Positional Encoding (Vaswani et al., 2017)

Sinusoidal Positional Encoding
PE(pos, 2i) = sin( pos / 10000^{2i/d_model} )
PE(pos, 2i+1) = cos( pos / 10000^{2i/d_model} )

pos = position in sequence (0, 1, 2, ..., n-1)
i = dimension index (0, 1, 2, ..., d_model/2 - 1)
d_model = embedding dimension (e.g. 512)

Added to token embeddings: input = token_embedding + PE
→ input has same shape as embedding: ℝ^{n × d_model}
  • ▸
    Why sinusoidal? Each position gets a unique encoding (different frequencies per dimension). More importantly, the relative position between two tokens can be expressed as a linear function of the PE — PE(pos+k) can be computed from PE(pos) linearly. This allows the model to attend to relative positions.
  • ▸
    Different frequencies: Low dimensions oscillate rapidly (fine-grained position). High dimensions oscillate slowly (coarse-grained position). Together they create a unique "fingerprint" for every position up to sequence length 10,000+.
  • ▸
    Learned positional encoding: Alternative approach — learn PE as parameters (like an embedding table). Used in BERT. Better performance on typical sequence lengths but cannot extrapolate to unseen lengths.
  • ▸
    Rotary Position Embedding (RoPE): Modern approach (GPT-NeoX, LLaMA). Encodes absolute position via rotation in complex space. Relative position information is preserved naturally in dot products. Better extrapolation to longer sequences than fixed PE.
  • ▸
    ALiBi (Attention with Linear Biases): Adds a linear bias to attention scores based on distance between tokens. No PE vectors added to embeddings — position is handled entirely in the attention score. Very efficient for long contexts.

Transformer Encoder Block

7.1 Encoder Block Architecture

  TRANSFORMER ENCODER BLOCK (one layer):

  Input: X  (n × d_model)  ← token embeddings + positional encoding
     │
  ┌──▼─────────────────────────────────────────────────────────────────┐
  │  SUBLAYER 1: Multi-Head Self-Attention                             │
  │                                                                    │
  │  Q = K = V = X  (self-attention: input attends to itself)         │
  │  MHA_output = MultiHead(X, X, X)          (n × d_model)           │
  └──┬─────────────────────────────────────────────────────────────────┘
     │
  ┌──▼─────────────────────────────────────────────────────────────────┐
  │  ADD & NORM 1  (Residual Connection + Layer Normalization)         │
  │  output1 = LayerNorm( X + MHA_output )                            │
  │  ← residual connection prevents vanishing gradient                │
  │  ← LayerNorm: normalize each token independently (per-sample)     │
  └──┬─────────────────────────────────────────────────────────────────┘
     │
  ┌──▼─────────────────────────────────────────────────────────────────┐
  │  SUBLAYER 2: Position-wise Feed-Forward Network (FFN)              │
  │                                                                    │
  │  FFN(x) = max(0, x·W₁ + b₁)·W₂ + b₂                             │
  │  W₁ ∈ ℝ^{d_model × d_ff}   d_ff = 2048 (4× d_model)              │
  │  W₂ ∈ ℝ^{d_ff × d_model}                                          │
  │  Applied INDEPENDENTLY to each position (hence "position-wise")   │
  └──┬─────────────────────────────────────────────────────────────────┘
     │
  ┌──▼─────────────────────────────────────────────────────────────────┐
  │  ADD & NORM 2                                                      │
  │  output2 = LayerNorm( output1 + FFN(output1) )                    │
  └──┬─────────────────────────────────────────────────────────────────┘
     │
  Output: (n × d_model)  → fed to next encoder layer
Figure 7.1 — Complete Transformer Encoder Block. Two sublayers: Multi-Head Self-Attention and FFN, each wrapped in Add&Norm.

7.2 Layer Normalization

  • ▸
    Formula: LayerNorm(x) = γ · (x − μ) / (σ + ε) + β. μ and σ are computed across the feature dimension (d_model) for each token independently. γ, β are learnable scale and shift parameters.
  • ▸
    Why LayerNorm not BatchNorm? Batch Normalization normalizes across the batch dimension — problematic for variable-length sequences (padding disrupts statistics). LayerNorm normalizes each token's feature vector independently — works the same for any batch size and sequence length.
  • ▸
    Pre-LN vs Post-LN: Original Transformer: Post-LN (LN applied after residual addition). GPT-2, BERT use Pre-LN (LN before each sublayer). Pre-LN is more stable for training very deep Transformers — gradient norms are better behaved.

7.3 Position-wise Feed-Forward Network (FFN)

  • ▸
    What it does: After multi-head attention mixes information across tokens, the FFN processes each token independently with the same MLP. It can be seen as a "memory" layer — storing factual associations (knowledge) in its weight matrices.
  • ▸
    Why 4× expansion? d_ff = 4 × d_model. Expanding to a larger inner dimension and projecting back creates a bottleneck that forces the FFN to learn compressed, efficient representations. The expansion provides capacity; the compression forces generalization.
  • ▸
    FFN as key-value memory: Research (Geva et al., 2021) shows each row of W₁ acts as a "key" that activates for certain input patterns, and the corresponding row of W₂ acts as the "value" — the information retrieved when that key activates. The FFN is an attention-like memory module.
  • ▸
    Modern activation functions: Original Transformer used ReLU. BERT/GPT-2 use GELU (Gaussian Error Linear Unit): GELU(x) = x·Φ(x) ≈ 0.5x(1 + tanh(√(2/π)(x + 0.044715x³))). Smooth, performs slightly better than ReLU in practice.

7.4 Residual Connections — Why Essential

  • ▸
    Gradient flow: output = LayerNorm(x + Sublayer(x)). The "+x" creates a direct path for gradients to flow: ∂loss/∂x = ∂loss/∂output · (1 + ∂Sublayer/∂x). The "1" term ensures gradients never vanish through depth.
  • ▸
    Representation preservation: Each sublayer only needs to learn the incremental change from the input — like ResNet. The sublayer output is added to the input, not replacing it. Early layers' representations persist through the entire network via the residual stream.
Section 8

Transformer Decoder Block

8.1 Decoder Block — Three Sublayers

The decoder block has three sublayers instead of the encoder's two. The extra sublayer enables the decoder to attend to the encoder's output (cross-attention).

  TRANSFORMER DECODER BLOCK:

  Inputs:
  • Y (decoder input, shifted right)  ← previously generated tokens
  • Encoder_output (from top encoder layer)

  ┌─────────────────────────────────────────────────────────────────┐
  │ SUBLAYER 1: Masked Multi-Head Self-Attention                   │
  │                                                                 │
  │ Q = K = V = Y  (decoder attends to itself)                     │
  │ CAUSAL MASK applied: token i cannot see tokens j > i           │
  │ Prevents "cheating" — cannot look at future target tokens      │
  └──────────┬──────────────────────────────────────────────────────┘
             │
  ┌──────────▼──────────────────────────────────────────────────────┐
  │ ADD & NORM 1:  output1 = LayerNorm(Y + MaskedMHA(Y,Y,Y))      │
  └──────────┬──────────────────────────────────────────────────────┘
             │
  ┌──────────▼──────────────────────────────────────────────────────┐
  │ SUBLAYER 2: Multi-Head Cross-Attention                         │
  │                                                                 │
  │ Q = output1                ← FROM DECODER (what do I need?)    │
  │ K = V = Encoder_output     ← FROM ENCODER (what does source    │
  │                              offer? what information is there?) │
  │ No mask — decoder can attend to ALL encoder positions          │
  └──────────┬──────────────────────────────────────────────────────┘
             │
  ┌──────────▼──────────────────────────────────────────────────────┐
  │ ADD & NORM 2:  output2 = LayerNorm(output1 + CrossAttn)        │
  └──────────┬──────────────────────────────────────────────────────┘
             │
  ┌──────────▼──────────────────────────────────────────────────────┐
  │ SUBLAYER 3: Position-wise FFN (same as encoder)                │
  └──────────┬──────────────────────────────────────────────────────┘
             │
  ADD & NORM 3 → output
Figure 8.1 — Transformer Decoder Block. Three sublayers: Masked Self-Attention, Cross-Attention (encoder-decoder), and FFN.
  • ▸
    Masked self-attention purpose: The decoder generates tokens autoregressively — one token at a time. During training, all target tokens are provided simultaneously for efficiency, but the mask ensures each position can only attend to past positions, preserving the autoregressive property.
  • ▸
    Cross-attention purpose: Q comes from the decoder (what the decoder needs at this step). K and V come from the encoder (what the source sentence offers). This is where the decoder "reads" the encoded source sentence. Different decoder positions attend to different source positions dynamically.
  • ▸
    Information flow: Masked self-attention lets each output token gather context from previously generated output tokens. Cross-attention then anchors that context to the relevant input information.

9.3 Training and Inference

  • ▸
    Training — teacher forcing: During training, the decoder receives the ground-truth target sequence (shifted right, starting with <SOS> token) at all positions simultaneously. The causal mask ensures causality. Loss: cross-entropy between predicted and true token at each position.
  • ▸
    Inference — autoregressive decoding: Generate one token at a time. Feed <SOS> → get token 1. Feed [<SOS>, token 1] → get token 2. Repeat until <EOS> or max length. Slower than training (sequential rather than parallel).
  • ▸
    Beam search: Instead of greedy decoding (always pick the most probable next token), maintain the top-k most probable partial sequences. Balances exploration and quality. Beam width k=4 is common.
  • ▸
    Label smoothing: Instead of hard 0/1 targets, use soft targets (1−ε for true class, ε/(V−1) for others). ε=0.1 in original Transformer. Prevents overconfidence, improves BLEU scores.
  • ▸
    Learning rate schedule: Warm up for warmup_steps = 4000 steps (linearly increase α), then decay as 1/√step. Formula: α = d_model^{−0.5} · min(step^{−0.5}, step · warmup_steps^{−1.5}). The warm-up prevents large early updates from destabilizing training.
Section 10

Self-Attention vs Cross-Attention

AspectSelf-AttentionCross-Attention
Q, K, V source All three come from the SAME sequence: Q=K=V=X (same input) Q comes from one sequence (decoder); K and V from another (encoder output)
What it computes How each token relates to all other tokens in the SAME sequence How decoder tokens relate to encoder tokens — bridges two different sequences
Used in Both encoder (bidirectional) and decoder (causal/masked) Decoder only (the middle sublayer); also in some encoder designs
Masking Encoder: no mask (full bidirectional). Decoder: causal mask (lower-triangular) No mask — decoder can attend to all encoder positions at all steps
Purpose Context-sensitive token representations — each token "sees" all others Alignment — learn which source positions are relevant for each target token
Attention pattern n_tokens × n_tokens matrix (square) n_target × n_source matrix (rectangular if different lengths)
Exam Point — Three Types of Attention in a Transformer

1. Encoder self-attention: Q=K=V from encoder input. No mask. Every source token attends to every other. Builds contextual representations of the input.

2. Decoder masked self-attention: Q=K=V from decoder input (target). Causal mask applied. Each target token attends only to previous target tokens. Maintains autoregressive property.

3. Encoder-decoder cross-attention: Q from decoder, K=V from encoder output. No mask. Each target token dynamically aligns to relevant source positions. This is the translation/alignment step.

Section 11

Vision Transformer (ViT) — Full Architecture

11.1 The Key Idea — Images as Sequences of Patches

ViT (Dosovitskiy et al., Google Brain — 2020) applies the standard Transformer encoder directly to images with minimal modifications. The key insight: split an image into a grid of fixed-size patches, flatten each patch into a vector, treat these patch vectors as a sequence of "tokens," and process with a standard Transformer encoder.

ViT's Radical Simplicity

No convolutional layers. No pooling. No inductive bias about local connectivity or translation equivariance. Just patch embeddings + positional encoding + standard Transformer encoder. When trained on large datasets (JFT-300M), ViT matches or exceeds state-of-the-art CNNs with less training compute. The inductive biases of CNNs (locality, translation equivariance) help with small datasets but become less important at large scale.

11.3 Key Components Explained

A. Patch Embedding

  • ▸
    Flattening: Each 16×16×3 patch has P²·C = 256×3 = 768 values. These are flattened into a 1D vector of size P²C = 768.
  • ▸
    Linear projection: A learnable weight matrix E ∈ ℝ^{(P²·C) × d_model} projects each flattened patch to d_model dimensions. Equivalent to a 16×16 Conv with stride 16 and d_model output channels — many implementations use Conv2d for efficiency.
  • ▸
    Why patches? Treating individual pixels as tokens would give 224×224 = 50,176 tokens — O(n²) attention cost would be infeasible (50,176² ≈ 2.5 billion). Patches of size 16×16 give 196 tokens — manageable.

B. [CLS] Token

  • ▸
    Purpose: A learnable vector prepended to the patch sequence. After passing through all transformer layers, its output representation aggregates global image information from all patches via self-attention. Used as the image-level representation for classification.
  • ▸
    Why not just average? Average pooling of all patch outputs discards relative position information. The [CLS] token can learn to selectively attend to and aggregate the most task-relevant information from all patches.
  • ▸
    Alternative: Global Average Pooling (GAP) over all patch outputs also works and avoids the CLS token's special role. DeiT and some ViT variants use GAP instead.

C. Positional Encoding in ViT

  • ▸
    1D learnable: Standard ViT uses 1D learnable positional embeddings — a 197×768 parameter matrix (one 768-dim vector per position). Simpler than 2D encodings; works well in practice.
  • ▸
    2D-aware sinusoidal: Encodes both row and column positions of each patch separately using sin/cos. Better for tasks requiring fine spatial reasoning.
  • ▸
    Interpolation for different resolutions: ViT trained at one resolution can be applied to other resolutions by interpolating the positional embeddings (bilinear interpolation in 2D grid). Important for fine-tuning at higher resolution than pre-training.

Section 12

ViT Variants and Improvements

Training Efficiency
DeiT — Data-efficient Image Transformers
Touvron et al., Facebook AI — 2020

Problem: ViT needs JFT-300M images to work well. Impractical for researchers without Google-scale data.

Solution: Knowledge distillation from a strong CNN teacher (RegNet). A distillation token is added alongside [CLS], trained to match the teacher's class prediction. Also: aggressive augmentation (RandAugment, CutMix), Mixup, repeated augmentation.

Result: DeiT-B achieves 81.8% top-1 on ImageNet trained on ImageNet-only in 3 days on 8 GPUs — matching ViT-B trained on JFT-300M.

Key innovation: Knowledge distillation + aggressive augmentation enables ViT without massive data
Hierarchical Processing
Swin Transformer
Liu et al., Microsoft — 2021

Problem: ViT processes all patches at the same resolution — expensive for dense prediction tasks (detection, segmentation). Also, fixed-resolution patch tokens can't capture multi-scale features.

Solution: Hierarchical patch merging (like CNN spatial pyramid). Shifted Window Attention — compute attention within local windows (7×7 patches), then shift windows by half to enable cross-window communication.

Result: Linear complexity O(n) instead of O(n²). Multi-scale features available. Swin-L: 87.3% ImageNet top-1. COCO SOTA for both detection and segmentation.

Key innovation: Local window attention + hierarchical feature maps = ViT meets CNN design
Self-Supervised ViT
DINO — Self-Distillation
Caron et al., Facebook AI — 2021

Core idea: Train ViT in a self-supervised manner using self-distillation with no labels. A student network is trained to match the output of a momentum teacher (exponential moving average of student). Applied to multiple augmented views of the same image.

Emergent property: Self-supervised DINO ViT features contain surprisingly rich object-level semantic information. Without any labels or fine-tuning, attention maps clearly segment objects from background — something no CNN achieves spontaneously.

Key result: Unsupervised segmentation quality emerges from self-supervised ViT training
Masked Pretraining
MAE — Masked Autoencoders
He et al., Facebook AI — 2021

Core idea: Mask 75% of image patches randomly. Train encoder-decoder to reconstruct the masked patches from visible 25%. Encoder only processes visible patches — computationally efficient. Decoder is lightweight and discarded after pre-training.

Why 75% works: Unlike BERT (15% masking), images have high spatial redundancy — 15% masking is trivially easy (nearby patches complete the missing ones). 75% forces genuine scene understanding. Reconstruction target: raw pixel values (not tokenized patches).

ViT-H fine-tuned from MAE: 87.8% top-1 on ImageNet. Better than supervised pre-training.
Part 4 — Large-Scale Pre-training with Transformers
Section 13

BERT — Bidirectional Encoder Representations from Transformers

13.1 Overview

BERT (Devlin et al., Google AI — 2018) was the watershed moment for NLP. It demonstrated that pre-training a deep bidirectional Transformer on massive unlabeled text, then fine-tuning on specific tasks with a small task-specific head, achieves state-of-the-art on 11 diverse NLP tasks simultaneously.

13.2 BERT Architecture

  • ▸
    Architecture: Standard Transformer Encoder only (no decoder). BERT-Base: L=12 layers, H=768, A=12 heads, ~110M parameters. BERT-Large: L=24, H=1024, A=16 heads, ~340M parameters.
  • ▸
    Input representation: Three embeddings summed: Token embedding + Segment embedding (sentence A or B) + Positional embedding. Tokens use WordPiece tokenization (subword units, ~30,000 vocabulary).
  • ▸
    Special tokens: [CLS] prepended to every sequence. [SEP] separates two sentences. [MASK] replaces masked tokens during pre-training.

13.3 BERT Pre-training Tasks

  TASK 1: MASKED LANGUAGE MODELING (MLM)

  Original: "The cat [MASK] on the mat and [MASK] purring"
  Goal: Predict the masked tokens ("sat", "was")

  Masking strategy for each selected token (15% of all tokens):
  • 80%: Replace with [MASK]     "sat" → [MASK]
  • 10%: Replace with random word "sat" → "walked"   (keeps D robust)
  • 10%: Keep original           "sat" → "sat"        (keeps R consistent)

  Why this split? If all masked tokens → [MASK], model learns to only
  predict when it sees [MASK], but at fine-tuning time there are no [MASK]
  tokens → mismatch. The 10/10 fractions reduce this train-test discrepancy.

  ─────────────────────────────────────────────────────────────────────
  TASK 2: NEXT SENTENCE PREDICTION (NSP)

  Input: [CLS] Sentence_A [SEP] Sentence_B [SEP]

  50% of the time: Sentence_B is the ACTUAL next sentence → label = IsNext
  50% of the time: Sentence_B is a RANDOM sentence       → label = NotNext

  [CLS] token output → Binary classifier
  Teaches model to understand sentence relationships
  (Used for QA, inference tasks that need cross-sentence reasoning)
Figure 13.1 — BERT's two pre-training objectives: Masked LM (predict masked tokens) and Next Sentence Prediction (predict sentence order).

13.4 BERT Fine-tuning

  • ▸
    Fine-tuning philosophy: Take the pre-trained BERT. Add a minimal task-specific head (one or two extra layers). Fine-tune ALL weights on the labeled task data. The same BERT parameters serve as initialization for all downstream tasks.
  • ▸
    Classification tasks: Use [CLS] output → Dense(num_classes) + Softmax. Fine-tune on sentiment, NLI, spam detection.
  • ▸
    Token-level tasks (NER, POS): Use each token's output → Dense(num_tags) + Softmax. Fine-tune on NER, POS tagging, word-level classification.
  • ▸
    Question Answering (SQuAD): Input: [CLS] question [SEP] passage [SEP]. Add two vectors S and E (learned). Score each token as answer start: S·hᵢ. Score each token as answer end: E·hᵢ. Fine-tune to find the span (start_i, end_j) in the passage that answers the question.
  • ▸
    Fine-tuning efficiency: With only ~3 epochs of fine-tuning (hours, not days), BERT surpassed previous SOTA on 11 NLP benchmarks — many by wide margins. This established fine-tuning as the dominant NLP paradigm.

Section 14

GPT Family — Autoregressive Language Modeling

14.1 GPT Architecture and Pre-training Objective

GPT (Generative Pre-trained Transformer, Radford et al., OpenAI — 2018) uses the Transformer decoder architecture (without cross-attention) and trains with a standard autoregressive language modeling objective: predict the next token given all previous tokens.

GPT Pre-training Objective
L(U) = Σᵢ log P(uᵢ | uᵢ₋k, ..., uᵢ₋₁; Θ)

Predict each token uᵢ from the previous k tokens (context window)
Maximize the conditional log-likelihood of each token given its left context
Θ = Transformer parameters trained by stochastic gradient descent

Key: CAUSAL (left-to-right) attention — each token attends only to previous tokens (future tokens masked). This makes the model naturally autoregressive.

14.2 GPT Evolution — GPT-1 to GPT-4

Generative Pre-training
GPT-1
Radford et al., OpenAI — 2018

Architecture: 12 Transformer decoder layers, d_model=768, 12 heads. 117M parameters. Trained on BooksCorpus (800M words, ~7,000 books).

Key insight: Unsupervised pre-training on large text corpus + supervised fine-tuning on task-specific labeled data outperforms task-specific models trained from scratch. Demonstrated generative pre-training for the first time.

First model to show: pre-train a Transformer decoder → fine-tune → strong NLP performance
Zero-Shot Transfer
GPT-2
Radford et al., OpenAI — 2019

Scale: 1.5B parameters (vs GPT-1's 117M). Trained on WebText: 40GB of high-quality Reddit links. 48 Transformer decoder layers for XL variant.

Key finding: When scaled up, zero-shot performance (no fine-tuning) on downstream tasks becomes competitive. The model begins to generalize from the pre-training distribution to task descriptions given in text. Initially withheld from public due to misuse concerns.

First demonstration of zero-shot task generalization from language model pre-training
Few-Shot In-Context Learning
GPT-3
Brown et al., OpenAI — 2020

Scale: 175B parameters. Trained on ~500B tokens (Common Crawl, BooksCorpus, Wikipedia, Books1/2, WebText). 96 layers, d_model=12,288, 96 heads.

Emergent ability: In-context learning (ICL) — provide a few examples in the prompt (few-shot), the model performs the task without any gradient updates. Few-shot GPT-3 matches or exceeds fine-tuned models on many benchmarks. Demonstrated emergent capabilities that simply weren't present at smaller scale.

Few-shot and zero-shot learning at scale. In-context learning as a new paradigm.
Instruction Following
InstructGPT / ChatGPT
Ouyang et al., OpenAI — 2022

RLHF (Reinforcement Learning from Human Feedback): 3-step process: (1) Supervised fine-tuning on demonstrations. (2) Train a reward model on human preference rankings. (3) Fine-tune GPT with PPO (Proximal Policy Optimization) to maximize reward.

Key result: InstructGPT with 1.3B params is preferred by humans over raw GPT-3 (175B) on most tasks — alignment matters more than raw scale. RLHF dramatically improves helpfulness, harmlessness, and honesty.

RLHF aligns LLMs with human preferences. Foundation of all modern assistant models.
Multimodal Reasoning
GPT-4
OpenAI — 2023

Multimodal: Accepts both text and image inputs. Achieves human-level performance on many professional exams (bar exam, GRE, AMC). Significantly improved over GPT-3.5 on complex reasoning, instruction following, and truthfulness.

Architecture: Undisclosed by OpenAI. Estimated 1T parameters (MoE). 128K context window (GPT-4-Turbo). Trained with improved RLHF and constitutional AI-like techniques.

State-of-the-art multimodal reasoning. Benchmark: 90th percentile on bar exam vs 10th for GPT-3.5
Open-Weight LLM
LLaMA / LLaMA 2 / LLaMA 3
Meta AI — 2023–2024

LLaMA 1: 7B–65B parameters. Trained on 1.4T tokens of public data. LLaMA-13B outperforms GPT-3 (175B) on most benchmarks — efficiency through data quality and model design (RMSNorm, RoPE, SwiGLU activation).

LLaMA 3: 8B–70B parameters. Trained on 15T tokens. Instruction-tuned variants match or beat GPT-4 on several benchmarks. Fully open-weight — spawned thousands of specialized fine-tuned models (Vicuna, Alpaca, Mistral).

Demonstrated: compute-optimal training with high-quality data > brute force scale

14.3 Emergent Abilities of Large Language Models

  • ▸
    Definition: Emergent abilities are capabilities that are NOT present or meaningful in smaller models but appear suddenly and sharply when model scale exceeds a certain threshold. They cannot be predicted by simply extrapolating from smaller model behavior.
  • ▸
    Chain-of-thought reasoning: Below ~100B parameters, chain-of-thought (CoT) prompting ("think step by step") does not help. Above ~100B, CoT dramatically improves multi-step arithmetic and logical reasoning.
  • ▸
    In-context learning: Few-shot examples in the prompt become effective guides for task performance at scale. Smaller models cannot effectively leverage this.
  • ▸
    Instruction following: Larger models can follow complex, nuanced natural language instructions without fine-tuning. Smaller models often ignore or misinterpret instructions.
  • ▸
    Calibration: Large models become better calibrated — their expressed confidence better matches their actual accuracy. Critical for trustworthy AI deployment.
Section 15

T5, CLIP and Other Foundation Models

15.1 T5 — Text-to-Text Transfer Transformer

  • ▸
    Core idea: Reframe EVERY NLP task as text-to-text. Classification, translation, summarization, QA — all formulated as: input a text string, output a text string. One single model with one single objective (maximum likelihood on text) for all tasks.
  • ▸
    Examples of task formulation: Translation: "translate English to German: The cat sat" → "Die Katze saß". Sentiment: "sst2 sentence: This movie is great" → "positive". QA: "question: What is the capital of France? context: Paris is..." → "Paris".
  • ▸
    Architecture: Full Transformer encoder-decoder. T5-Small (60M) to T5-11B. Pre-trained on C4 (Colossal Clean Crawled Corpus — 750GB of cleaned Common Crawl). Span corruption pre-training objective: mask spans of consecutive tokens; predict them.
  • ▸
    Impact: T5's text-to-text framing influenced all subsequent encoder-decoder models (FLAN-T5, UL2, mT5). Showed that a unified text generation objective is sufficient for diverse NLP tasks.

15.2 CLIP — Contrastive Language-Image Pre-training

  CLIP Architecture (Radford et al., OpenAI — 2021):

  Training data: 400 million (image, text) pairs from the internet
  (e.g., image of a dog + caption "a fluffy golden retriever playing fetch")

  ┌──────────────────┐          ┌──────────────────────┐
  │  Image Encoder   │          │   Text Encoder       │
  │  (ViT or ResNet) │          │   (Transformer)      │
  └────────┬─────────┘          └──────────┬───────────┘
           │ image_embedding (512-dim)      │ text_embedding (512-dim)
           └──────────────┬─────────────────┘
                          │
              Contrastive Loss (InfoNCE):
              • Same (image,text) pair → embeddings should be CLOSE
              • Different pairs → embeddings should be FAR apart

  For N (image, text) pairs in a batch:
  • N positive pairs (diagonal of similarity matrix) → maximize similarity
  • N×(N-1) negative pairs (off-diagonal) → minimize similarity
Figure 15.1 — CLIP training. Image and text encoders jointly trained to align matching pairs via contrastive loss.
  • ▸
    Zero-shot classification: To classify an image into classes ["cat", "dog", "car"]: compute image embedding. Compute text embeddings of "a photo of a cat", "a photo of a dog", "a photo of a car". Predict the class whose text embedding is most similar to the image embedding. No task-specific training needed.
  • ▸
    Why CLIP works: Internet image-text pairs encode a vast amount of visual concepts, relationships, and attributes. The contrastive objective forces the encoders to align visual and linguistic representations of the same concept.
  • ▸
    Applications: Zero-shot classification, image-text retrieval, visual question answering backbone, text-to-image generation (DALL-E uses CLIP embeddings to guide generation). CLIP embeddings are widely used as visual features in downstream models.
  • ▸
    CLIP limitations: Weak at fine-grained classification (distinguishing dog breeds). Struggles with counting, spatial reasoning, and abstract concepts. Performance degrades on distributions far from internet image-text pairs.

16.2 The Scaling Laws

  • ▸
    Kaplan et al. scaling laws (OpenAI, 2020): Model performance scales as a power law with (1) model parameters N, (2) training tokens D, and (3) compute budget C. Each of these independently and predictably improves performance — no saturation observed at current scales.
  • ▸
    Chinchilla scaling laws (Hoffmann et al., DeepMind, 2022): For a given compute budget, you should scale model size and training tokens equally. GPT-3 (175B, 300B tokens) was undertrained. Optimal: Chinchilla (70B parameters, 1.4T tokens) outperforms GPT-3 on most benchmarks with 2.5× fewer parameters. Data quality × quantity matters as much as model size.
  • ▸
    Compute-optimal training: N_optimal ≈ C^{0.5} (parameters scale as square root of compute). D_optimal ≈ C^{0.5} (data scales similarly). Doubling compute should double both parameters and tokens, not double just one.
  • ▸
    Implication: LLaMA-13B trained on 1T tokens outperforms GPT-3 (175B) trained on 300B tokens. Training smaller models on more tokens is more efficient than training large models for less time.

Exam Point — LoRA (Low-Rank Adaptation) Explained

LoRA freezes all pre-trained weights and injects low-rank decomposition matrices into the attention weight matrices. For weight matrix W ∈ ℝ^{d×d}: instead of fine-tuning W directly, learn ΔW = A·B where A ∈ ℝ^{d×r} and B ∈ ℝ^{r×d} with r ≪ d (typically r=4 to 64). At inference: W_new = W + A·B. Training only updates A and B — typically 0.01–1% of total parameters. Achieves comparable performance to full fine-tuning with 10,000× fewer trainable parameters. Standard approach for fine-tuning LLaMA, GPT, BERT on custom domains.

Module 3 — Complete Summary

  • QKV
    Q, K, V: Q = what am I looking for? K = what do I offer? V = actual information I carry. All three produced by learned linear projections of the input. Decoupling K and V allows separate "search" and "retrieve" representations.
  • AP
    Attention Pooling: Output = softmax(score(Q,K)) · V. Soft, differentiable lookup over all values weighted by Q-K compatibility. Scores: additive (Bahdanau), dot-product, scaled dot-product (÷√d_k), bilinear. Scale prevents softmax saturation for large d_k.
  • SDA
    Scaled Dot-Product Attention: Attention(Q,K,V) = softmax(QKᵀ/√d_k)·V. Shapes: Q(n_q×d_k), K(n_k×d_k), V(n_k×d_v) → output (n_q×d_v). Causal masking sets upper triangle to −∞ for autoregressive generation.
  • MHA
    Multi-Head Attention: h parallel attention heads, each with own W^Q_i, W^K_i, W^V_i (d_model/h dims). Outputs concatenated and projected by W^O. Each head learns different relationship types. O(n²·d) complexity — bottleneck for long sequences.
  • PE
    Positional Encoding: PE(pos, 2i) = sin(pos/10000^{2i/d}). Added to token embeddings. Enables relative position computation. ViT uses learned 1D PE; modern LLMs use RoPE or ALiBi.
  • ENC
    Encoder Block: MH Self-Attention → Add&Norm → FFN (expand d_ff=4×d_model, ReLU/GELU) → Add&Norm. Pre-LN placement (before sublayer) preferred in modern models for stability.
  • DEC
    Decoder Block: Masked Self-Attention → Add&Norm → Cross-Attention (Q from decoder, K,V from encoder) → Add&Norm → FFN → Add&Norm. 3 sublayers vs encoder's 2.
  • ViT
    Vision Transformer: Image → 16×16 patches → flatten → linear project → [CLS]+position encode → Transformer encoder → [CLS] output → classify. 196 patches for 224×224 image. No Conv inductive bias. Requires large data but scales better than CNNs.
  • VV
    ViT Variants: DeiT (knowledge distillation enables ImageNet-only training), Swin (local windows + hierarchical, linear complexity), DINO (self-supervised, emergent segmentation), MAE (mask 75% patches, reconstruct).
  • BRT
    BERT: Encoder-only. MLM (predict 15% masked tokens, 80/10/10 strategy) + NSP pre-training. [CLS] for classification; token outputs for sequence labeling; span prediction for QA. Same weights fine-tuned for all tasks.
  • GPT
    GPT Family: Decoder-only, causal LM (next token prediction). GPT-2: zero-shot generalization. GPT-3 (175B): few-shot in-context learning. InstructGPT: RLHF aligns with human preferences. LLaMA: compute-optimal open-weight models.
  • FND
    Foundation Models: T5 (text-to-text, all NLP as generation), CLIP (contrastive image-text alignment, zero-shot classification), DALL-E (text-to-image). Scaling laws: model and data should scale together (Chinchilla). LoRA enables efficient fine-tuning of frozen large models.
