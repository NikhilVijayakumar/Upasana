Section 1

VGGNet — Depth Through Uniformity

1.1 Background and Motivation

VGGNet (Simonyan & Zisserman, Oxford Visual Geometry Group — 2014) explored a fundamental question: if we keep the architecture simple and uniform, how much does increasing depth help? Their answer was definitive — depth is the critical factor, even with the simplest possible building blocks.

  • ▸
    ILSVRC 2014 result: VGG-16 achieved 7.3% top-5 error on ImageNet. VGG-19 achieved 7.1%. Came 2nd to GoogLeNet but became far more widely used due to its simplicity and transferability.
  • ▸
    Core philosophy: Use only 3×3 Conv (smallest possible, captures all 8-directional spatial context), stride 1, same padding. Use 2×2 MaxPool stride 2 for downsampling. Nothing else. No inception modules, no asymmetric convolutions.
  • ▸
    Variants: VGG-11 (8 conv + 3 FC), VGG-13, VGG-16, VGG-19. Number indicates total weight layers. VGG-16 is the most widely used.

1.2 The 3×3 Filter Argument — Why Stacking Works

Core Insight — Small Filters Stacked Are Better Than Large Filters Alone

Two 3×3 layers → effective 5×5 receptive field. Uses 2×9 = 18 weights + 2 non-linearities.

One 5×5 layer → same 5×5 field. Uses 25 weights + 1 non-linearity.

Three 3×3 layers → effective 7×7 field. Uses 3×9 = 27 weights vs 49 for one 7×7.

Result: Stacked small filters are (1) cheaper in parameters, (2) have more non-linearities, (3) act as implicit regularizers. VGG proved this experimentally at scale.

1.4 Parameter Analysis and Bottleneck

  • ▸
    Total parameters: ~138 million. Breakdown: All 13 Conv layers together ≈ 14.7M. FC6 alone: 7×7×512×4096 = 102.7M. FC7: 4096×4096 = 16.8M. FC8: 4.1M. FC layers hold 89% of all parameters.
  • ▸
    Pattern — double channels, halve spatial: Each pooling halves H and W (224→112→56→28→14→7). Each new block doubles channel depth (64→128→256→512). This is the fundamental VGG design pattern.
  • ▸
    Memory at inference: ~528 MB for FP32 weights. The FC layers are the bottleneck — modern networks replace them with Global Average Pooling (GAP), saving 100M+ parameters.
  • ▸
    VGG as feature extractor: Remove FC layers; use Pool5 output (7×7×512) or any intermediate feature map as a rich image descriptor. Used in style transfer, detection, and countless downstream tasks.
Exam Point — VGG Key Facts

1. Only 3×3 Conv and 2×2 MaxPool — no other layer types in the feature extractor. 2. Channels double at each block: 64→128→256→512→512. 3. Spatial dims halve at each pool: 224→112→56→28→14→7. 4. 138M params; 89% in 3 FC layers. 5. VGG-16 = 13 Conv + 3 FC = 16 weight layers. 6. Strong transfer learning backbone despite age.

Section 2

ResNet — Residual Learning for Very Deep Networks

2.1 The Degradation Problem

Simply adding more layers to VGG-style networks did not keep helping after ~20 layers. Surprisingly, adding layers made training accuracy worse — not because of overfitting (the training loss itself was higher), but because very deep networks were harder to optimize with standard backpropagation. Gradients vanished through too many layers.

He et al.'s Insight — The Degradation Problem

If a 56-layer network is strictly harder to optimize than a 20-layer network, there is something wrong with how deeper networks learn. A 56-layer net should be at least as good as a 20-layer net — it could theoretically just set the extra 36 layers to identity mappings. But in practice, networks cannot learn identity mappings through long chains of non-linear transformations. The solution: explicitly build identity mappings as shortcut connections.

2.2 The Residual Block — Core Innovation

  PLAIN BLOCK (VGG-style):          RESIDUAL BLOCK (ResNet):

  Input: x                          Input: x ─────────────────────┐
     │                                  │                          │ skip
  [Conv 3×3 + BN + ReLU]           [Conv 3×3 + BN + ReLU]        │ connection
     │                                  │                          │ (identity)
  [Conv 3×3 + BN + ReLU]           [Conv 3×3 + BN]               │
     │                                  │                          │
  Output: H(x)                     [    + x    ] ◀────────────────┘
                                        │
                                    [  ReLU  ]
                                    Output: H(x) = F(x) + x

  Network learns H(x) directly.    Network learns residual F(x) = H(x) − x.
                                    Then H(x) = F(x) + x.
Figure 2.1 — Plain block vs Residual block. The skip connection (shortcut) allows gradients to flow directly to earlier layers.

2.3 Why Residual Learning Works — Three Perspectives

  • ▸
    Perspective 1 — Easier optimization: If the optimal function is close to identity (H(x) ≈ x), it is much easier to push F(x) → 0 (a small perturbation from zero) than to learn H(x) ≈ x through stacked non-linear transformations. The residual formulation reframes what needs to be learned into a simpler target.
  • ▸
    Perspective 2 — Gradient highway: During backpropagation, gradients flow through the skip connection directly: ∂L/∂x = ∂L/∂(F+x) = ∂L/∂F + 1. The "+1" ensures that even if the residual branch gradient ∂L/∂F is tiny (vanishing), a gradient of 1 still flows through the identity path. Deep networks can now receive non-zero gradient signals at early layers.
  • ▸
    Perspective 3 — Ensemble of shallow networks: Residual networks can be unrolled as an exponential ensemble of shallow networks of different depths. This gives them ensemble-like properties — robustness and good generalization.

2.4 Residual Block Variants

  BASIC BLOCK (ResNet-18, 34):      BOTTLENECK BLOCK (ResNet-50, 101, 152):

  Input: x (C channels)             Input: x (256 channels)
     │                                  │
  [3×3 Conv, C filters, BN, ReLU]  [1×1 Conv, 64 filters, BN, ReLU]  ← reduce channels
     │                                  │
  [3×3 Conv, C filters, BN]         [3×3 Conv, 64 filters, BN, ReLU]  ← spatial conv
     │           ┌── x ──┐              │
  [ + ] ◀────────┘        │         [1×1 Conv, 256 filters, BN]        ← restore channels
  [ReLU]                  │             │           ┌── x ──┐
  Output                  │         [ + ] ◀─────────┘        │
                          │         [ReLU]                    │
                  Identity or        Output               1×1 Conv
                  1×1 Conv           (when dim changes)   projection
Figure 2.2 — Basic block (2 Conv layers) vs Bottleneck block (3 Conv layers with 1×1 compression). Bottleneck enables deeper nets with same compute.
  • ▸
    Basic block: Two 3×3 Conv layers. Used in ResNet-18 and ResNet-34. Straightforward.
  • ▸
    Bottleneck block: 1×1 Conv (reduce channels: 256→64) → 3×3 Conv (spatial processing at lower dim) → 1×1 Conv (expand channels: 64→256). Much cheaper computationally: 1×1+3×3+1×1 vs 3×3+3×3. Enables ResNet-50, 101, 152 without excessive compute.
  • ▸
    Projection shortcut: When input and output dimensions differ (e.g., spatial halving or channel doubling), the skip connection uses a 1×1 Conv projection to match dimensions. Otherwise it is a pure identity (no parameters).
  • ▸
    Batch Normalization placement: Standard: Conv → BN → ReLU → Conv → BN → ADD → ReLU. Pre-activation ResNet (He et al., 2016): BN → ReLU → Conv → BN → ReLU → Conv → ADD. Pre-activation performs slightly better and simplifies analysis.

2.6 ResNet Architecture — Stage Structure

  • ▸
    Stage 0 — Stem: 7×7 Conv, 64 filters, stride 2 → 112×112. BN + ReLU. 3×3 MaxPool stride 2 → 56×56×64. Large initial Conv captures broad patterns quickly.
  • ▸
    Stage 1 (conv2_x): 56×56, 64 channels. No spatial downsampling. Residual blocks (basic or bottleneck). First stage sets the feature base.
  • ▸
    Stage 2 (conv3_x): 28×28, 128 channels. First block uses stride 2 for spatial halving + projection for channel doubling.
  • ▸
    Stage 3 (conv4_x): 14×14, 256 channels. Deepest stage — most residual blocks. Core of representational learning.
  • ▸
    Stage 4 (conv5_x): 7×7, 512 channels (for ResNet-18/34) or 2048 channels (for ResNet-50+).
  • ▸
    Head: Global Average Pooling (GAP) → 1×1×C → Dense(1000) + Softmax. GAP replaces FC layers — only 1000 parameters in the classification head regardless of backbone depth.

2.7 ResNet Variants and Successors

  • ▸
    Wide ResNet (WRN): Wider (more channels) but shallower. WRN-28-10 (28 layers, 10× wider) outperforms ResNet-1001 with 8× fewer parameters. Width sometimes > depth.
  • ▸
    ResNeXt: Introduces grouped convolutions (cardinality). Replaces one large Conv with many parallel small Convs with shared weights. ResNeXt-50 32×4d: 32 groups of 4-channel Convs. Better accuracy for same params.
  • ▸
    DenseNet: Every layer connects to ALL subsequent layers (not just next). Feature reuse is maximum. But memory-intensive — requires storing all intermediate feature maps.
  • ▸
    SE-Net (Squeeze-and-Excitation): Adds a channel attention module to each residual block. Squeezes spatial info via GAP, excites channels with learned weights. Won ILSVRC 2017.
Exam Point — Why ResNet Succeeded Where VGG Failed for Deep Networks

VGG: stacks plain layers → gradient vanishes through 16+ layers → degradation problem. ResNet: adds identity skip connections → gradient highway bypasses vanishing → can train 152+ layers. The key equation is H(x) = F(x) + x. The gradient ∂L/∂x = ∂L/∂F + 1 ensures the "+1" term always provides a non-zero gradient signal regardless of depth.

Section 3

EfficientNet — Principled Network Scaling

3.1 The Scaling Problem

After ResNet proved depth is crucial, researchers kept scaling networks — wider, deeper, higher resolution input. But each axis was scaled independently and arbitrarily. EfficientNet (Tan & Le, Google Brain — 2019) asked: is there a principled way to scale CNNs that maximizes accuracy for a given compute budget?

3.2 The Three Scaling Dimensions

  THREE INDEPENDENT SCALING DIMENSIONS:

  1. DEPTH (d):       more layers       → larger receptive field, more complex features
                      ResNet-50 → 101 → 152 (same width, more layers)

  2. WIDTH (w):       more channels     → richer representations per layer
                      VGG-64 → 128 → 256 channels (same depth, wider)

  3. RESOLUTION (r):  larger input image → finer spatial detail
                      224×224 → 300×300 → 600×600 (same network, larger input)

  ──────────────────────────────────────────────────────────────────────
  Problem: scaling only ONE dimension has diminishing returns quickly.
  Scaling them TOGETHER but UNBALANCED also suboptimal.
  EfficientNet: scale ALL THREE jointly in a principled ratio.
Figure 3.1 — Three scaling dimensions. EfficientNet scales all three simultaneously using compound coefficients.

3.3 Compound Scaling — The Core Formula

Compound Scaling Coefficients
Given a target compute budget φ (compound coefficient):

depth: d = αᵠ
width: w = βᵠ
resolution: r = γᵠ

Subject to: α · β² · γ² ≈ 2 ← FLOPS double every time φ increases by 1
α ≥ 1, β ≥ 1, γ ≥ 1

Found via grid search on small baseline (EfficientNet-B0):
α = 1.2, β = 1.1, γ = 1.15
  • ▸
    Why β² and γ²? FLOPS scale as O(d·w²·r²). Doubling width quadruples FLOPS (w²). Doubling resolution also quadruples FLOPS (r²). Doubling depth only doubles FLOPS (d). So the constraint α·β²·γ²≈2 ensures each unit increase in φ exactly doubles total FLOPS.
  • ▸
    Intuition: Higher resolution inputs need deeper networks to process fine-grained details and wider networks to capture more patterns at each level. The three dimensions must grow in sync — they are not independent.
  • ▸
    Grid search procedure: Fix φ=1. Search α, β, γ satisfying the constraint. Apply these optimal ratios as: d=α^φ, w=β^φ, r=γ^φ for any target φ.

3.4 MBConv — The Building Block of EfficientNet

EfficientNet uses MBConv (Mobile Inverted Bottleneck Convolution) as its basic block, borrowed from MobileNetV2. This is the key to its parameter efficiency.

  MBConv Block (with SE — Squeeze and Excitation):

  Input: x (C channels)
      │
  [1×1 Conv — Expand: C → t×C channels]     ← expansion factor t (e.g. 6)
  [BN + Swish activation]
      │
  [3×3 or 5×5 Depthwise Conv — t×C groups]  ← spatial filtering (no channel mixing)
  [BN + Swish]
      │
  [SE Block: GAP → FC(t×C/r) → FC(t×C) → sigmoid → scale features] ← channel attention
      │
  [1×1 Conv — Project: t×C → C_out channels] ← compress back
  [BN (no activation)]
      │
  [+ x (skip if C_in == C_out)]               ← residual connection (when dims match)
Figure 3.2 — MBConv block with Squeeze-and-Excitation. Uses depthwise separable conv for efficiency + SE for channel attention.
  • ▸
    Swish activation: f(x) = x · σ(x). Smoother than ReLU (differentiable everywhere). Outperforms ReLU on EfficientNet. Also called SiLU (Sigmoid Linear Unit).
  • ▸
    Inverted bottleneck: Unlike ResNet (wide→narrow→wide), MBConv goes narrow→wide→narrow. Expansion happens inside the depthwise conv step — hence "inverted."
  • ▸
    SE block (Squeeze-Excitation): Global Average Pooling squeezes spatial info → two FC layers produce per-channel weights (Excitation) → scale each channel by its learned weight. Adds channel attention: "which features matter for this image?"
  • ▸
    Why efficient: Depthwise Conv has C_in parameters per spatial filter (no cross-channel mixing). Standard Conv has C_in×C_out×f×f. For C=128, f=3: depthwise = 128×9 = 1152 vs standard = 128×128×9 = 147,456. Over 100× fewer parameters for spatial filtering.

1. Problem solved: Arbitrary/one-dimensional scaling of CNNs (only depth, or only width) is suboptimal. All three dimensions interact. 2. Same accuracy as GPipe with 8× fewer parameters.

Section 5

Object Detection — Fundamentals

5.1 Core Concepts

  • ▸
    Bounding Box: Defined by (x_center, y_center, width, height) or (x_min, y_min, x_max, y_max). The spatial localization output for each detected object.
  • ▸
    Anchor boxes: Pre-defined boxes of multiple sizes and aspect ratios placed at each position in the feature map. Detectors predict offsets from these anchors rather than absolute coordinates — makes the regression problem easier.
  • ▸
    IoU (Intersection over Union): IoU = Area(A ∩ B) / Area(A ∪ B). Measures overlap between predicted and ground-truth box. IoU ≥ 0.5 = correct detection (PASCAL VOC). IoU thresholds 0.5:0.05:0.95 averaged = COCO AP metric.
  • ▸
    Non-Maximum Suppression (NMS): Multiple anchors may fire for the same object. NMS: (1) Sort all predictions by confidence. (2) Keep highest-confidence box. (3) Remove all other boxes with IoU > threshold with it. (4) Repeat. Keeps exactly one box per object.
  • ▸
    Feature Pyramid Network (FPN): Multi-scale feature extraction. A top-down pathway with lateral connections merges deep (semantic) features with shallow (spatial) features at multiple resolutions. Essential for detecting objects at vastly different scales.
IoU Formula and Confidence Score
IoU(A, B) = |A ∩ B| / |A ∪ B| = |A ∩ B| / (|A| + |B| − |A ∩ B|)

Detection confidence = P(object) × IoU(pred, ground_truth)

mAP = (1/C) × Σ_c AUC(Precision-Recall curve for class c)

6.3 Key Components — Detailed

A. Backbone + FPN

  • ▸
    ResNet-50/101 backbone: Extracts hierarchical feature maps C2 (1/4 res), C3 (1/8), C4 (1/16), C5 (1/32) from input image.
  • ▸
    FPN top-down pathway: Starts from C5 (high semantics, low resolution). Upsamples 2× and adds (lateral connection) with C4, C3, C2. Each level gets 256 channels via 1×1 Conv. Produces P2–P5 multi-scale feature maps with both rich semantics AND spatial precision.
  • ▸
    Multi-scale detection: Small anchors are matched to P2 (high resolution). Large anchors to P5 (low resolution). Each level handles its appropriate scale range.

B. Region Proposal Network (RPN)

  • ▸
    Operation: A 3×3 sliding window Conv over each FPN feature map. At each position, predicts for k=9 anchors: (1) binary objectness (object or background), (2) 4 box offset values (dx, dy, dw, dh).
  • ▸
    Anchor definition: At each feature map position: 3 scales (32², 64², 128² pixels) × 3 aspect ratios (1:1, 1:2, 2:1) = 9 anchors. Anchors are aligned to input image coordinates.
  • ▸
    Training: Anchors with IoU ≥ 0.7 with any ground-truth box → positive. IoU < 0.3 → negative. Others → ignored. Loss = binary cross-entropy (objectness) + smooth L1 (box offsets) for positives only.

C. RoIAlign — The Critical Fix

  • ▸
    Problem with RoIPool: RoIPool quantizes (rounds) the RoI boundaries to discrete feature map cells. This introduces misalignment of up to half a cell — devastating for pixel-level mask prediction.
  • ▸
    RoIAlign solution: Divides the RoI into k×k bins (typically 7×7 for detection, 14×14 for masks). Within each bin, samples 4 points using bilinear interpolation at exact float coordinates — no rounding. Results are max-pooled per bin.
  • ▸
    Impact: Mask AP improved by 3–5% simply by switching from RoIPool to RoIAlign, without any other change. Critical for precise mask boundaries.

D. Mask Head

  • ▸
    Architecture: Takes 14×14×256 RoIAlign output. Applies 4 × (3×3 Conv, 256 channels, ReLU). One transposed Conv (deconv) upsamples 14×14 → 28×28. Final 1×1 Conv → C×28×28 binary masks (one per class). Sigmoid activation.
  • ▸
    Class-specific masks: Predicts a separate 28×28 binary mask for each of C classes. At inference, only the mask for the predicted class is selected. This decouples mask prediction from classification — the mask head doesn't need to worry about which class it is, just the shape.
  • ▸
    Mask loss: Average binary cross-entropy loss only over the ground-truth class mask. Computed only for positive (foreground) RoIs.

6.4 Multi-Task Loss

Mask R-CNN Total Loss
L_total = L_rpn_cls + L_rpn_box + L_cls + L_box + L_mask

L_rpn_cls: binary cross-entropy (objectness) ← RPN
L_rpn_box: smooth L1 (anchor box regression) ← RPN
L_cls: cross-entropy (C+1 classes) ← classification head
L_box: smooth L1 (box refinement) ← box head
L_mask: avg binary cross-entropy (28×28 mask) ← mask head

All 5 losses are summed with equal weight (λ=1 for each)
Exam Point — What Makes Mask R-CNN Unique

1. Three heads: Classification + Box regression + Instance mask — all sharing the same backbone and RoI features. 2. RoIAlign: Bilinear interpolation preserves spatial alignment — crucial for pixel-precise masks. 3. Class-specific masks: Predicts C binary masks per RoI; selects only the predicted class's mask. Decouples "what" from "where exactly." 4. FPN backbone: Multi-scale features handle objects at vastly different sizes.

Section 7

YOLO — Evolution from v1 to v4

7.1 YOLO Philosophy — You Only Look Once

The core YOLO insight: object detection as a single regression problem. Rather than two stages (propose → classify), YOLO predicts bounding boxes and class probabilities directly from the full image in one forward pass. This makes it dramatically faster — enabling real-time detection.

YOLOv1
Single-Pass Grid Detection
Redmon et al., CVPR 2015 — 45 FPS on Titan X

Core idea: Divide image into S×S grid (7×7). Each cell predicts B=2 bounding boxes (x,y,w,h,confidence) and C=20 class probabilities. Output tensor: 7×7×(2×5+20) = 7×7×30.

Loss: Sum of squared errors across all predictions. Penalizes localization, confidence, and classification errors. Weighted to handle class imbalance (λ_coord=5, λ_noobj=0.5).

Limitations: struggles with small objects, objects in clusters, unusual aspect ratios. Each cell predicts only 1 class — cannot detect two classes in same cell.
YOLOv2 / YOLO9000
Better, Faster, Stronger
Redmon & Farhadi, CVPR 2017 — 67 FPS

Anchor boxes: Replaces arbitrary box predictions with learned anchor shapes (k-means clustering on training set bounding boxes). 5 anchors per cell. Predicts offsets from anchors, not absolute coordinates. Much better localization.

Batch Normalization: Added to every Conv layer (removed Dropout). Darknet-19 backbone (19 Conv layers). Multi-scale training: randomly resizes input every 10 batches (320 to 608).

YOLO9000: trained jointly on ImageNet (9000 classes) + COCO using hierarchical softmax (WordTree). Can detect 9000 object categories.
YOLOv3
Multi-Scale Detection with Darknet-53
Redmon & Farhadi, arXiv 2018 — 30 FPS (608×608)

Multi-scale predictions: Detects at 3 scales (52×52, 26×26, 13×13 feature maps). Large objects on coarse map, small on fine map. 3 anchors per scale = 9 anchors total.

Darknet-53 backbone: 53 Conv layers with residual shortcuts. Combination of ResNet-style connections + DarkNet design. Better feature extraction than Darknet-19.

Logistic classifiers: Replaced Softmax with independent logistic classifiers for multi-label detection (an object can be "person" AND "woman").

Most widely deployed YOLO variant in industry. Good balance of speed and accuracy. 51.5% mAP on COCO at 51 FPS.
YOLOv4
Systematic Bag-of-Freebies
Bochkovskiy et al., 2020 — 62 FPS (608×608)

Bag of Freebies (BoF): Training tricks that improve accuracy at no inference cost: CutMix/Mosaic augmentation, DropBlock regularization, CIoU loss, label smoothing, self-adversarial training.

Bag of Specials (BoS): Small inference-time additions: Mish activation, SPP (Spatial Pyramid Pooling), PANet (Path Aggregation Network neck), SAM attention.

CSPDarknet53 backbone: Cross Stage Partial connections — splits feature map and processes only half through the dense block, merging results. Reduces FLOPS while maintaining accuracy.

43.5% AP on COCO at 65 FPS on RTX 2080Ti. Comprehensive engineering effort — systematic evaluation of all modern tricks.

7.2 YOLO Prediction Format

YOLO Box Prediction (per anchor at grid cell i,j)
Box center (relative to cell):
bx = σ(tx) + cx ← σ ensures bx stays within cell
by = σ(ty) + cy ← cx, cy = cell top-left coordinates

Box dimensions (relative to anchor):
bw = pw · e^tw ← pw = anchor width (from k-means)
bh = ph · e^th ← exponential ensures positive size

conf = σ(to) ← P(object) × IoU(pred, gt)

Class probabilities (YOLOv3):
P(class_i) = σ(tc_i) ← independent logistic, not softmax

YOLOv5 — Architecture in Depth

8.1 Overview and Context

YOLOv5 (Glenn Jocher, Ultralytics — 2020) is not a direct successor to YOLOv4 by the original authors (Redmon). It is an independent re-implementation and extension in PyTorch. Despite controversy about the version naming, YOLOv5 became enormously popular due to its ease of use, extensive documentation, and excellent speed-accuracy tradeoff.

Why YOLOv5 Became the Industry Standard

YOLOv5 was the first YOLO fully implemented in PyTorch (previous versions used Darknet C framework). Ultralytics provided an end-to-end training/evaluation/deployment pipeline, detailed documentation, easy model export (ONNX, TensorRT, CoreML), and active maintenance. This made it the go-to detector for industry and research from 2020 onwards.

8.3 CSP (Cross Stage Partial) Connection

  • ▸
    Idea: Split input feature map into two parts along channels. One part goes through the dense block (residual layers). The other bypasses directly. Both are concatenated at the end and projected.
  • ▸
    Benefit: Reduces computation: the expensive residual block operates on only half the channels. Also reduces gradient duplication — the direct path and the residual path share gradients differently, improving learning efficiency.
  • ▸
    vs. ResNet: ResNet adds x to F(x). CSP concatenates x' (half channels) with F(x'') (other half through residual). Both reduce vanishing gradients but CSP is more compute-efficient.

8.4 PANet Neck

  • ▸
    FPN limitation: FPN top-down path: information flows from deep to shallow layers. Feature at P2 has traveled a long path from P5 — long-range connections may weaken semantic information.
  • ▸
    PANet (Path Aggregation Network): Adds a second bottom-up path after FPN's top-down path. So information can now travel both: (1) deep (semantic) → shallow (spatial), AND (2) shallow (spatial) → deep (semantic). Maximum feature propagation between all levels in just 2 hops.
  • ▸
    Result: Richer feature maps at all scales — better detection of objects at all sizes. Standard neck in most modern YOLO variants.

8.5 YOLOv5 Loss Function

YOLOv5 Total Loss
L = λ_box · L_box + λ_obj · L_obj + λ_cls · L_cls

L_box: CIoU loss (Complete IoU) ← penalizes center distance + aspect ratio
L_obj: Binary Cross-Entropy ← objectness per anchor
L_cls: Binary Cross-Entropy ← class probabilities (multi-label)

Weights: λ_box=0.05, λ_obj=1.0, λ_cls=0.5

8.6 CIoU Loss — Improvement Over Simple IoU

  • ▸
    IoU Loss: L = 1 - IoU. Problem: if boxes don't overlap at all, IoU=0 and gradient=0 — the box cannot move toward the ground truth.
  • ▸
    GIoU (Generalized IoU): Adds a term for the smallest enclosing box: L = 1 − IoU + |C \ (A∪B)| / |C|. Provides gradient even when boxes don't overlap.
  • ▸
    DIoU (Distance IoU): Adds distance between box centers: L = 1 − IoU + ρ²(b,b_gt)/c². Also used in NMS (DIoU-NMS) — better than standard NMS for occluded objects.
  • ▸
    CIoU (Complete IoU): Adds aspect ratio consistency: L = 1 − IoU + ρ²/c² + αv, where v = (4/π²)(arctan(w_gt/h_gt) − arctan(w/h))² and α = v/(1−IoU+v). Simultaneously optimizes overlap, center distance, AND aspect ratio.

Section 9

YOLOv7 — State-of-the-Art Real-Time Detection

9.1 Background

YOLOv7 (Wang, Bochkovskiy, Liao — 2022, from the original YOLOv4 team) is the direct successor to YOLOv4. It introduced several architectural innovations under the concept of "trainable bag-of-freebies" — optimizations that improve accuracy during training without adding inference cost.

YOLOv7 Core Claim

At the time of publication (July 2022): YOLOv7 surpassed all known real-time object detectors in both speed and accuracy at 5–160 FPS on COCO. YOLOv7-E6 achieved 56.8% AP on COCO — better than previous detectors at the same speed. YOLOv7-tiny runs at 160 FPS on V100 GPU. "State of the art" from 5 FPS to 160 FPS range.

9.2 E-ELAN — Efficient Layer Aggregation Network

  • ▸
    Problem with stacking CSP: As networks get deeper, gradient flow and feature reuse become suboptimal. ELAN (Extended Efficient Layer Aggregation Network) controls the shortest and longest gradient paths by grouping conv layers into blocks with selective skip connections.
  • ▸
    E-ELAN (Extended ELAN): YOLOv7's extension. Uses group convolution for channel expansion, shuffles and merges cardinality (multiple parallel computation paths). Does NOT destroy the original gradient path — the extended network block maintains the ability to learn and converge.
  • ▸
    Effect: Better feature reuse, stronger gradient flow, higher parameter efficiency. The backbone design is more principled than ad-hoc CSP stacking.

9.3 Model Re-parametrization — RepConv

  • ▸
    Concept: Use a more complex structure (multiple parallel branches) during training to learn richer representations. At inference, mathematically merge all branches into a single Conv — no extra cost at deployment time. "Trainable freebies."
  • ▸
    RepConv structure (training): Three parallel branches: 3×3 Conv + 1×1 Conv + identity shortcut. Each branch has its own BN. Trained with gradients flowing through all three.
  • ▸
    RepConv (inference): Absorb all branches into one 3×3 Conv by fusing BN parameters and adding branch kernels. Single Conv operation = fast inference, no multi-branch overhead. Same accuracy as multi-branch training.
  • ▸
    Why it works: At training time, the 1×1 and identity branches provide additional gradient paths and implicit regularization. At inference, the structure collapses to a compact form with the same expressive power.

9.4 Auxiliary Heads and Lead Head

  • ▸
    Lead head: The final detection output used at inference time.
  • ▸
    Auxiliary heads: Additional detection heads attached to intermediate layers. During training only — their losses help train the backbone more effectively (similar to Inception's auxiliary classifiers but more principled).
  • ▸
    Lead head guided label assignment: The auxiliary heads use the lead head's predictions to generate their training labels — not the raw ground truth. This ensures auxiliary heads and lead head are consistent, improving overall training quality.

Section 11

Object Tracking — Problem Setup

11.1 Detection vs Tracking

Object detection answers: "What objects are in this frame, and where?" — applied independently to each frame.

Object tracking answers: "Which detected object in frame t is the same object as which detection in frame t+1?" — it assigns consistent IDs to objects across time.

  • ▸
    Multi-Object Tracking (MOT): Track multiple objects simultaneously. Each track has a unique ID maintained across all frames. Output: trajectory of every object over time.
  • ▸
    Online vs Offline tracking: Online: processes frames sequentially in real-time — must make association decisions immediately. Offline: can look at future frames — can use global optimization (Hungarian algorithm over entire video). DeepSORT is online.
  • ▸
    Challenges: Occlusion (object hidden behind another). ID switch (tracker assigns wrong ID after occlusion). Appearance change (different lighting, viewpoint). Crowded scenes (many similar-looking objects close together).

11.2 Tracking-by-Detection Paradigm

The dominant modern approach. Decouple detection and tracking:

11.3 SORT — Simple Online and Realtime Tracking

SORT (Bewley et al., 2016) is the foundation of DeepSORT. It uses only two classical algorithms for tracking: Kalman Filter for prediction and Hungarian Algorithm for association.

  • ▸
    Kalman Filter: Estimates the current state of a track (position and velocity) given noisy measurements. Predicts where the object will be in the next frame. Corrects prediction when a new detection is associated. Assumes constant velocity motion model.
  • ▸
    State vector: [u, v, s, r, u̇, v̇, ṡ] where (u,v) = bounding box center, s = scale (area), r = aspect ratio, and the dots represent velocities. Constant aspect ratio assumed (ṙ=0).
  • ▸
    Hungarian Algorithm: Optimal assignment algorithm. Given a cost matrix between N detections and M existing tracks, finds the minimum-cost assignment in O(n³) time. Used to solve the bipartite matching problem: which detection should be assigned to which existing track?
  • ▸
    Association cost: In SORT: IoU between predicted track bounding box and detection bounding box. Low IoU = high cost. Tracks with IoU < 0.3 with all detections are left unassigned.
  • ▸
    SORT limitation: Uses only IoU for association — purely geometric. When objects are occluded or move fast, IoU-based association fails. Appearance is completely ignored → many ID switches in crowded scenes.
Section 12

DeepSORT — Deep Appearance Descriptor + SORT

12.1 Core Innovation

DeepSORT (Wojke, Bewley, Paulus — 2017) extends SORT by adding a deep appearance descriptor — a small CNN that embeds each detected bounding box into a 128-dimensional feature vector. Association now considers both geometric (IoU) similarity AND visual appearance similarity, dramatically reducing ID switches.

DeepSORT = SORT + Deep Appearance Features

SORT fails when IoU cannot distinguish tracks (occlusion, fast motion, crowded scenes). DeepSORT's appearance descriptor asks: "Does this detection look like the same person/object as that track, regardless of where the boxes are?" This visual memory allows re-identification after long occlusions — something pure geometric tracking cannot do.

12.3 Components Explained in Depth

A. Appearance Descriptor CNN

  • ▸
    Architecture: Small CNN (e.g., 8 Conv layers, Batch Norm, ELU activations). Input: 64×128×3 (cropped and resized bounding box). Output: 128-dimensional L2-normalized feature vector.
  • ▸
    Training: Pre-trained on person re-identification datasets (Market-1501, DukeMTMC — ~1000 different pedestrian identities). Uses softmax classification loss + triplet loss to learn discriminative appearance embeddings.
  • ▸
    Gallery: For each active track, maintain a gallery of the last 100 appearance descriptors. When computing appearance cost, compare new detection to ALL gallery embeddings and take the minimum distance (best match).
  • ▸
    Cosine distance: dist(a, b) = 1 − (aᵀb)/(‖a‖·‖b‖). Range [0, 2]. Since embeddings are L2-normalized: dist = 1 − aᵀb. Threshold 0.2 — if cosine distance > 0.2, appearance is too different for association.

B. Kalman Filter in DeepSORT

  • ▸
    State space: x = [u, v, γ, h, u̇, v̇, γ̇, ḣ]ᵀ. u,v = box center. γ = aspect ratio. h = height. Velocities estimated from temporal differences.
  • ▸
    Prediction step: x̂ₜ = F·xₜ₋₁ (state transition). Pₜ = F·Pₜ₋₁·Fᵀ + Q (covariance update). F = state transition matrix (constant velocity). Q = process noise (model uncertainty).
  • ▸
    Update step (when matched): Kalman gain K = Pₜ·Hᵀ·(H·Pₜ·Hᵀ + R)⁻¹. Updated state: xₜ = x̂ₜ + K·(zₜ − H·x̂ₜ). Updated covariance: Pₜ = (I − K·H)·Pₜ. H = observation matrix. R = measurement noise. zₜ = new detection measurement.
  • ▸
    Mahalanobis distance: d²(k,i) = (dᵢ − ŷₖ)ᵀ Sₖ⁻¹ (dᵢ − ŷₖ). Accounts for prediction uncertainty via Sₖ = H·Pₖ·Hᵀ + R. A detection far from the predicted position (in uncertainty-normalized space) gets a high cost — acts as a gate.

C. Cascaded Matching

  • ▸
    Two-stage matching: DeepSORT uses cascaded matching rather than a single Hungarian assignment. First, try to match tracks that disappeared most recently (highest priority). Tracks that have been missing longer have less reliable Kalman predictions.
  • ▸
    Stage 1: Match tracks that were updated last frame (age=1). These have the most reliable predictions.
  • ▸
    Stage 2: Remaining unmatched tracks (age=2, 3, ...) are matched to remaining unmatched detections using IoU only (Kalman prediction degrades for older tracks).

Section 13

Tracking Metrics, SORT vs DeepSORT, and Applications

13.1 Multi-Object Tracking Metrics

  • ▸
    MOTA (Multi-Object Tracking Accuracy): MOTA = 1 − (FP + FN + IDSW) / GT. Combines false positives, false negatives, and ID switches. Main tracking accuracy metric. Higher is better.
  • ▸
    MOTP (Multi-Object Tracking Precision): Average IoU between matched predictions and ground truth. Measures localization precision of matched tracks. Higher is better.
  • ▸
    IDF1 (ID F1 Score): Harmonic mean of ID precision and ID recall. Measures how long trajectories are correctly tracked under the same ID. More sensitive to ID switches than MOTA.
  • ▸
    IDSW (ID Switches): Number of times a tracked object changes its assigned ID. Main failure mode of SORT. DeepSORT significantly reduces IDSW by using appearance features.
  • ▸
    Mostly Tracked (MT) / Mostly Lost (ML): MT = % of ground truth trajectories that are tracked for at least 80% of their lifespan. ML = % tracked for less than 20%. Higher MT and lower ML is better.

13.4 Applications of YOLO + DeepSORT

Pedestrian Tracking

Surveillance cameras in malls, airports, stations. Count people, detect crowd density, identify suspicious behavior. DeepSORT maintains IDs through occlusions in crowds.

Sports Analytics

Track players and ball across video frames. Compute player trajectories, heatmaps, speed, distance covered. Used in football, basketball, tennis analytics.

Autonomous Driving

Track items on conveyor belts, count defects, monitor worker safety (PPE compliance). Real-time tracking ensures every item is accounted for.

Wildlife Monitoring

Track individual animals across camera trap footage. Count populations, study migration. Re-ID helps when animals temporarily leave the frame.

Exam Point — DeepSORT in 5 Points

Module 1 — Complete Summary

  • 1.
    VGGNet: 16 weight layers, only 3×3 Conv + 2×2 MaxPool. Channel pattern: 64→128→256→512. 138M params (89% in 3 FC layers). Two stacked 3×3 = 5×5 receptive field with fewer params. Strong transfer learning backbone despite large size.
  • 2.
    ResNet: Residual block H(x)=F(x)+x. Skip connection creates gradient highway (∂L/∂x includes "+1" term). Basic block (2×3×3) for ResNet-18/34; Bottleneck (1×1→3×3→1×1) for ResNet-50+. GAP replaces FC layers. Enables 152-layer training without degradation.
  • 3.
    EfficientNet: Compound scaling: d=α^φ, w=β^φ, r=γ^φ with α·β²·γ²≈2. MBConv building block: expand (1×1) → depthwise spatial (3×3/5×5) → SE attention → project (1×1). Swish activation. B0 (5.3M) matches ResNet-50 (25.6M). B7 achieves 84.3% top-1.
  • 4.
    Mask R-CNN: Extends Faster R-CNN. Backbone (ResNet+FPN) → RPN proposals → RoIAlign (bilinear, no quantization) → 3 parallel heads: classification, box regression, instance mask (28×28 binary, class-specific). 5-loss training. RoIAlign critical for pixel-precise masks.
  • 5.
    YOLO Evolution: v1 (grid regression), v2 (anchors + BN), v3 (Darknet-53, 3-scale), v4 (CSPDarknet + BoF/BoS). All use: anchor boxes, NMS, multi-scale feature maps, 3-part architecture (backbone → neck → head).
  • 6.
    YOLOv5: PyTorch, CSPDarknet53 backbone, SPP + PANet neck, Focus layer, 5 size variants (n/s/m/l/x), CIoU loss. Industry-standard choice. 156 FPS for small model.
  • 7.
    YOLOv7: E-ELAN backbone (principled feature aggregation), RepConv (multi-branch train → single Conv at inference), auxiliary training heads with lead-guided label assignment, implicit knowledge. 161 FPS, 51.4% COCO AP.
  • 8.
    DeepSORT: SORT (Kalman + Hungarian + IoU) + deep appearance CNN (128-dim ReID embedding). Cost = λ·cosine_appearance + (1-λ)·Mahalanobis_motion + gating. Cascaded matching. Reduces ID switches 61%. Track lifecycle: tentative → confirmed → deleted.
